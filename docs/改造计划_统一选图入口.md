## ② 统一 `openIconPicker` 入口函数

**目标**：将现有 3 个入口函数合并为 1 个，统一参数签名和选择器 HTML 模板。

### 2.1 新增函数

- [ ] `openIconPicker(ctx)` — 统一入口，`ctx.mode` 覆盖 5 种场景
- [ ] `renderIconPickerHTML(ctx)` — 统一选择器 HTML 模板
- [ ] `selectIcon(item, ctx)` — 统一分发，基于 `DISPATCH` 配置表

### 2.2 `ctx.mode` 5 种场景

| mode | 场景 | 涉及 ctx 参数 |
|------|------|-------------|
| `info_card_title` | info_card 板块标题图标 | `bi` |
| `desc_row` | 描述行图片 | `bi`, `di` |
| `icon_group` | 图标组行内图标 | `bi`, `rowDi`, `ii` |
| `base_row` | 部落基地存储行图标 | `si`, `bi`, `ri` |
| `base_item` | 部落基地物品图标 | `si`, `bi`, `ri`, `ci`, `ii` |

### 2.3 DISPATCH 配置表

- [ ] 定义 `DISPATCH = { info_card_title: {...}, desc_row: {...}, icon_group: {...}, base_row: {...}, base_item: {...} }`
- [ ] 每条目含 `path(ctx)` / `key` / `hasColor`（全部 `true`）
- [ ] `selectIcon` 统一处理：URL 写入 + 反色 mode + 亮度检测 + 视觉更新 + 关闭选择器

### 2.4 删除旧函数

- [ ] 删除 `openIconPicker_ic(bi)`
- [ ] 删除 `openIconPicker_desc(el, bi, di)`
- [ ] 删除原 `openIconPicker(si, bi, ri, ci, ii)`

### 2.5 更新调用点

- [ ] info_card 标题图标 → `openIconPicker({mode:'info_card_title', bi:N})`
- [ ] 描述行图片 → `openIconPicker({mode:'desc_row', bi:N, di:N})`
- [ ] 图标组行内图标 → `openIconPicker({mode:'icon_group', bi:N, rowDi:N, ii:N})`
- [ ] 部落基地存储行 → `openIconPicker({mode:'base_row', si:N, bi:N, ri:N})`
- [ ] 部落基地物品 → `openIconPicker({mode:'base_item', si:N, bi:N, ri:N, ci:N, ii:N})`

### 2.6 其他适配

- [ ] `composeIcons` 分发改为按 `ctx.mode` 匹配（替代 onclick 字符串匹配）
- [ ] 选择器按钮行全功能（搜索/组合/刷新/反色/清空），不区分模式

### 2.7 验证

- [ ] 5 种场景各测试一次选图 + 保存 + 预览渲染
- [ ] HTML 空行检查（0 连续空行）
- [ ] MD5 验证：4 路径 lovelace 一致
