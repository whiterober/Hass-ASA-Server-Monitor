# ASA 三层着色逻辑对比报告

> 生成时间: 2026-07-14 | 版本: v1339

---

## 1. 架构概览

| 层 | 位置 | 渲染引擎 | CSS 注入方式 |
|----|------|---------|------------|
| **编辑区** | asa-admin.html (JS) | 浏览器 DOM + inline style | `<style>` 标签 + 内联 |
| **预览区** | build_lovelace.py → preview_server.py | Python 生成 HTML | `<style>` 标签 |
| **实页** | HA lovelace | HA 前端渲染 | card_mod 注入 Shadow DOM |

---

## 2. 颜色系统

### 2.1 色条/色块颜色

| 属性 | 编辑区 | 预览区 / 实页 |
|------|--------|-------------|
| 默认值 | `auto` | `auto` |
| 默认渲染 | `.cat-color-dot.auto-theme`（outline 空心圆） | 无特殊样式（跟随主题） |
| 自定义色 | `background: <hex>` 内联 | `border-left-color: <hex>` |
| 透明度 | `.cat-opacity-dot` + `opacity` 内联 | 不单独支持透明度 |

**差异**: 编辑区有独立的透明度和颜色选择器（opacity dot + color dot），实页仅支持颜色。

### 2.2 引用块色条 (quote_color)

| 层 | 实现 |
|----|------|
| 编辑区 | `mcICQuoteColor{N}` hidden input + `.cat-color-dot` 选择器 |
| 预览区/实页 | `border-left: 4px solid <color>` 或 `var(--text)` auto 模式 |

## 3. 图片反色系统

### 3.1 亮度检测 → 反色分类

| 亮度 | CSS 类 | Dark 主题 | Light 主题 |
|------|--------|----------|-----------|
| `> 0.5` | `ic-auto-light` | 无 filter | `filter: invert(1)` |
| `≤ 0.5` | `ic-auto-dark` | `filter: invert(1)` | 无 filter |
| `null / 0.5` | `ic-auto-color` | 跟随 `--primary-text-color` | 跟随 `--primary-text-color` |

### 3.2 三层实现对比

| 层 | 亮度检测方式 | 反色 CSS |
|----|------------|---------|
| **编辑区** | JS `getImageLuminance()` 异步回调 → 写入 `image_native_luminance` | 内联 `class="ic-auto-dark/light/color"` |
| **预览区** | Python 读取 `image_native_luminance` → 生成 class | `<style>` 中 `[data-theme="dark"] .ic-auto-dark { filter: invert(1) }` |
| **实页** | 同预览区（build_lovelace.py 生成） | card_mod 注入同上 CSS |

**差异**: 编辑区 JS 异步计算，可能有时序问题（`_pendingBrightnessChecks` 轮询）。预览/实页直接读已存值。

### 3.3 图标组反色

| 层 | 特殊处理 |
|----|---------|
| 编辑区 | 每个 `ig-item` 独立检测亮度 |
| 预览区 | CSS `[data-theme] .ig-img.ic-auto-dark { filter: invert(1) }` |
| 实页 | 同上（card_mod），注意无 `ha-card` 前缀（v1159 修复） |

## 4. 服务器归属 → 线性态/色块态

### 4.1 状态定义

| server_states 值 | 含义 | CSS 类 |
|:--:|------|--------|
| 0 | 不显示 | `display: none` |
| 1 | **线性态** | `ic-linear-{serverId}` |
| 2 | **色块态** | `ic-block-{serverId}` |

### 4.2 渲染规则

| 层 | 线性态 | 色块态 |
|----|--------|--------|
| **编辑区** | `ic-linear-{sid}` class，无特殊样式 | `ic-block-{sid}` class，无特殊样式 |
| **预览区/实页** | `border-left: 4px solid {server_color}` | `background: {server_color} !important; color: {text_color}` |

### 4.3 颜色来源

| 来源 | 编辑区 | 预览区/实页 |
|------|--------|-----------|
| 服务器颜色 | `SERVER_MAP[sid].color` | `SERVER_MAP[sid].color` |
| 文字颜色 | N/A（不渲染） | `SERVER_MAP[sid].text_color` (白/黑) |
| 生成方式 | JS `SERVER_LIST` 内联 | Python `make_ic_css()` 动态生成 per-server CSS |

## 5. 地图筛选

### 5.1 编辑区

```javascript
// map_filter block: 生成 server 标签按钮
// 点击标签 → 设置 brSelectedServer → reRender()
// 其他板块: filter_maps 属性控制地图归属
```

### 5.2 预览区/实页

```css
/* 动态生成 per-server filter CSS */
.filter-label[data-map=Isl] { background: {color}; color: {bg}; }
.filter-radio:checked → 显示/隐藏 .filterable[data-filter-maps*=Isl]
```

**差异**: 编辑区用 JS 变量 `brSelectedServer`，预览/实页用纯 CSS radio button hack。

## 6. Dark/Light 主题

### 6.1 编辑区

| 组件 | Dark | Light |
|------|------|-------|
| 整体 | `<html data-theme="dark">` | `<html data-theme="light">` |
| 颜色变量 | `--text: #e0e0e0` 等 | `--text: #1a1a2e` 等 |
| 图片反色 | `[data-theme="dark"] img.ic-auto-dark { invert(1) }` | `[data-theme="light"] img.ic-auto-light { invert(1) }` |
| 切换方式 | 按钮 onclick `toggleTheme()` | 按钮 onclick `toggleTheme()` |

### 6.2 预览区

| 组件 | Dark | Light |
|------|------|-------|
| 来源 | URL param `?theme=dark` 或默认 | URL param `?theme=light` |
| 反色 | 同编辑区 CSS | 同编辑区 CSS |

### 6.3 实页 (HA)

| 组件 | Dark | Light |
|------|------|-------|
| 来源 | HA 主题 (Material You) | HA 主题 |
| 反色 | card_mod CSS 同预览区 | card_mod CSS 同预览区 |
| 变量 | `--primary-background-color` 等 HA 变量 | 同 |

## 7. 描述行渲染

### 7.1 字段对比

| 字段 | 编辑区 | 预览区/实页 |
|------|:--:|:--:|
| 文字 | ✅ 输入框 | ✅ `<span>` |

### 7.2 ×N 数量样式

|----|------|------|
| 编辑区 | 内联 `font-size` 动态计算 | - |

## 9. 关键差异汇总

| 差异点 | 编辑区 | 预览/实页 | 影响 |
|--------|--------|----------|------|
| 透明度 | ✅ | ❌ | 预览可能偏色 |
| 颜色选择器 | color dot + opacity dot | 无独立 UI | - |
| 地图筛选 | JS `brSelectedServer` | CSS radio hack | 行为等价 |
| 内联样式 | 大量 inline style | card_mod CSS | Shadow DOM 限制 |
| 图片加载 | 异步亮度检测 | 直接读已存值 | 时序差异 |
| 主题切换 | JS toggle | URL param / HA 主题 | - |

---

> **结论**: 编辑区和预览/实页的核心着色逻辑一致（同源 `SERVER_MAP` + `make_ic_css`），差异主要在 UI 交互层（颜色选择器、亮度检测时序）和 CSS 注入方式（inline vs card_mod）。
