# ASA 三层着色逻辑对比报告

> 生成时间: 2026-07-14 | 版本: v1339

---

## 1. 架构概览

| 层 | 位置 | 渲染引擎 | CSS 注入方式 |
|----|------|---------|------------|
| **编辑区** | asa-admin.html (JS) | 浏览器 DOM + inline style | `<style>` 标签 + 内联 |
| **预览区** | build_lovelace.py → preview_server.py | Python 生成 HTML | `<style>` 标签 |
| **实页** | HA lovelace | HA 前端渲染 | card_mod 注入 Shadow DOM |

> 🗣️ 白话：编辑区是你操作的网页，预览区是点保存后生成的静态页面，实页是 Home Assistant 最终展示的样子。三层的颜色逻辑是一样的，但注入 CSS 的方式不同——编辑区直接写，实页要通过 HA 的 card_mod 插件塞进去。

## 2. 颜色系统

### 2.1 色条/色块颜色

| 属性 | 编辑区 | 预览区 / 实页 |
|------|--------|-------------|
| 默认值 | `auto` | `auto` |
| 默认渲染 | `.cat-color-dot.auto-theme`（outline 空心圆） | 无特殊样式（跟随主题） |
| 自定义色 | `background: <hex>` 内联 | `border-left-color: <hex>` |

**差异**: 编辑区通过圆点选择器改色 → 保存 → 预览/实页左边竖线（`border-left: 4px solid <color>`）变色，效果一致。

> 🗣️ 白话：每个板块左边那条竖线的颜色，编辑区里点小圆点选色，保存后预览和实页都会变。auto 就是跟随主题色，选红色竖线就变红。

### 2.2 引用块色条 (quote_color)

| 层 | 实现 |
|----|------|
| 编辑区 | `mcICQuoteColor{N}` hidden input + `.cat-color-dot` 选择器 |
| 预览区/实页 | `border-left: 4px solid <color>` 或 `var(--text)` auto 模式 |

> 🗣️ 白话：就是上面说的左边竖线。编辑区用隐藏输入框存颜色值，预览和实页直接把它渲染成一条 4px 宽的竖线。

## 3. 图片反色系统

### 3.1 亮度检测 → 反色分类

| 亮度 | CSS 类 | Dark 主题 | Light 主题 |
|------|--------|----------|-----------|
| `> 0.5` | `ic-auto-light` | 无 filter | `filter: invert(1)` |
| `≤ 0.5` | `ic-auto-dark` | `filter: invert(1)` | 无 filter |
| `null` | `ic-auto-dark` + toast 报错 | `filter: invert(1)`（兜底） | 无 filter（兜底） |

### 3.2 三层实现对比

| 层 | 亮度检测方式 | 反色 CSS |
|----|------------|---------|
| **编辑区** | JS `getImageLuminance()` 异步回调 → 写入 `image_native_luminance` | 内联 `class="ic-auto-dark/light/color"` |
| **预览区/实页** | 同一套：Python 读取已存值 → `make_ic_css()` 生成 CSS | `<style>` / card_mod 注入同源 CSS |

> 🗣️ 白话：深色图片在暗主题下看不清，需要反色。编辑区用 JS 选图时立刻算亮度，预览和实页直接读之前存好的值。CSS 是同一套，预览和实页完全一致。

### 3.3 图标组反色

| 层 | 特殊处理 |
|----|---------|
| 编辑区 | 每个 `ig-item` 独立检测亮度 |
| 预览区/实页 | 同一套 CSS（`make_ic_css()` 生成），反色规则均无 `ha-card` 前缀 |

> 🗣️ 白话：图标组里每个小图标也支持反色。预览和实页用的是 build_lovelace.py 里同一个函数生成的同一套 CSS，完全一致。

## 4. 服务器归属 → 线性态/色块态

### 4.1 状态定义

| server_states 值 | 含义 | CSS 类 |
|:--:|------|--------|
| 0 | 不显示 | `display: none` |
| 1 | **线性态** | `ic-linear-{serverId}` |
| 2 | **色块态** | `ic-block-{serverId}` |

### 4.2 编辑区：descServerPopup 弹窗

编辑区通过描述行右侧的服务器按钮（`🗺` 图标）触发弹窗，内含三层内容：

**第一层：11 个服务器按钮**（孤岛～俱乐部）

| 状态 | 视觉 | 含义 |
|:--:|------|------|
| 0 (off) | 暗色空心圆边框（opacity 0.35） | 不归属此服务器 |
| 1 (linear) | 亮色空心圆 + `box-shadow` 光环 | 线性态 |
| 2 (block) | 实心填充背景（服务器色） | 色块态 |

> 点击循环切换：0 → 1 → 2 → 0。同一描述行**单选**（只能激活一个服务器），清零后回退到 `_default` 线性态。

**第二层：分隔线**

**第三层：4 个预设按钮**（`FIXED_STYLES`）

| ID | 标签 | 颜色 | MDI 图标 |
|----|------|------|---------|
| `_default` | 默认 | `auto`（跟随主题） | `mdi-palette` |
| `_hint` | 提示 | `#2196F3` 蓝 | `mdi-information` |
| `_warn` | 警告 | `#FF9800` 橙 | `mdi-alert` |
| `_remark` | 备注 | `#757575` 灰 | `mdi-comment-text-outline` |

> 预设同样三态循环，`_default` 永不回 0（0→1→2→1 锁底）。

**联动效果**：选中服务器/预设后，`selectDescServer()` 自动把描述行文字颜色（`mcICDescColor`）设为对应的服务器/预设颜色；清零时恢复 `auto`。按钮上的徽章（`updateDescSrvBadge`）同步更新样式——暗色=off、空心光环=linear、实心填充=block。

### 4.3 预览区/实页：CSS 渲染

**编辑区**：仅存数据（`server_states` hidden input），不渲染任何视觉 class。

**预览区/实页** — 同源 CSS，按元素类型效果如下：

#### 色块态 `ic-block-{sid}`

| 元素 | CSS 选择器 | 效果 |
|------|----------|------|
| 描述行整体 | `.ic-text.ic-block-{sid}` | 背景 = 服务器颜色；文字变白；`border-radius:6px;padding:2px 6px` |
| ×N 角标 | `.ic-text.ic-block-{sid} .ic-qty` | 绝对定位右下角；白字 + 无色背景 + `-webkit-text-stroke:2px` 用服务器色描边 |
| MDI 图标 | `.ic-text.ic-block-{sid} .mdi, ha-icon` | 颜色变白 |
| 描述行大图 | `.ic-block-img` | 绝对定位右浮 30×30px；父元素 `padding-right:34px` 给它留空间 |
| 空心徽章 | `[class*="ic-block-"] .ic-badge-hollow` | 半透白底 + 白字；auto-block 模式反转为半透文字色底 + 文字色 |
| 整个板块 | `.info-card-block.ic-block-{sid}` | 背景 = 服务器色 8% opacity（极淡） |
| 图标组 | `.ic-text.ic-block-{sid}`（包裹 wrapper） | 整行背景染色 + 文字变白；标题去徽章用纯文字；隐藏分隔线 |
| 默认 `_default` | `.ic-text.ic-block-_default` | `color-mix(--primary-text-color 10%, transparent)`，文字不反色 |

#### 线性态 `ic-linear-{sid}`

| 元素 | CSS 选择器 | 效果 |
|------|----------|------|
| 徽章 | `.ic-text.ic-linear-{sid} .ic-badge` | 底色 = 服务器色 15% opacity；文字 = 服务器色 |
| MDI 图标 | `.ic-linear-{sid} .mdi, ha-icon` | 颜色 = 服务器色 |
| 图标组标题徽章 | `.ig-row-wrapper.ic-linear-{sid} .ig-title-badge` | 同徽章规则 |
| 图标组分隔线 | `.ig-row-wrapper.ic-linear-{sid} .ig-title-line` | 颜色 = 服务器色，40% opacity |
| `::after` 伪元素 | `[class*="ic-linear-"]::after` | `display:none`（隐藏） |

> 🗣️ 白话：色块态会把整行文字背景染色、文字变白、×N 角标挪到右下角、MDI 图标变白、大图右浮、图标组整行染色且标题去徽章。线性态则温和得多——只染徽章和 MDI 图标的颜色，图标组的标题徽章和分隔线也跟着变色，图标组之间的默认分割线隐藏。

### 4.4 颜色来源

§4.2 中所有"服务器色"来自 `SERVER_MAP`：

| 来源 | 值 | 用途 |
|------|-----|------|
| `SERVER_MAP[sid].color` | `#4CAF50` 等 | 色块态：描述行背景、×N 描边；线性态：徽章/图标/分隔线颜色 |
| 色块态文字 | `var(--primary-background-color)` | 跟随主题（暗主题白字、亮主题黑字） |
| 编辑器 | 只存 JSON（`server_states:{"Isl":2}`） | 不参与渲染 |

> 同一个服务器颜色，在色块态做背景、在 LINEAR 态做点缀。文字颜色直接复用主题变量，没有独立配置。

## 5. Dark/Light 主题

### 5.1 编辑区

| 组件 | Dark | Light |
|------|------|-------|
| 整体 | `<html data-theme="dark">` | `<html data-theme="light">` |
| 颜色变量 | `--text: #e0e0e0` 等 | `--text: #1a1a2e` 等 |
| 图片反色 | `[data-theme="dark"] img.ic-auto-dark { invert(1) }` | `[data-theme="light"] img.ic-auto-light { invert(1) }` |
| 切换方式 | 按钮 onclick `toggleTheme()` | 按钮 onclick `toggleTheme()` |

### 5.2 预览区

| 组件 | Dark | Light |
|------|------|-------|
| 来源 | URL param `?theme=dark` 或默认 | URL param `?theme=light` |
| 反色 | 同编辑区 CSS | 同编辑区 CSS |

### 5.3 实页 (HA)

| 组件 | Dark | Light |
|------|------|-------|
| 来源 | HA 主题 (Material You) | HA 主题 |
| 反色 | card_mod CSS 同预览区 | card_mod CSS 同预览区 |
| 变量 | `--primary-background-color` 等 HA 变量 | 同 |

> 🗣️ 白话：暗色/亮色切换。编辑区点右上角月亮/太阳按钮切换，预览区通过 URL 参数控制，实页跟随 HA 的主题设置。深色图片在暗主题下自动反色变亮，浅色图片在亮主题下反色变暗。

## 6. 描述行渲染

### 6.1 字段对比

| 字段 | 编辑区 | 预览区/实页 |
|------|:--:|:--:|
| 文字 | ✅ 输入框 | ✅ `<span>` |
| 文字颜色 | ✅ color dot + opacity dot | ✅ inline `style="color:xxx"`（非 block）/ `background:xxx`（block） |
| 加粗 | ✅ B 按钮 → `.ic-bold` | ✅ `.ic-bold` |
| 图片 | ✅ 图标选择器 → `.ic-desc-img` | ✅ `<img>` |
| 数量 ×N | ✅ qty-btn 选择器 → `.ic-qty` | ✅ `.ic-qty` badge |
| 服务器归属 | ✅ server btn + descServerPopup | ✅ `data-server-states` + `ic-linear-`/`ic-block-` class |

### 6.2 ×N 数量样式

| 层 | 实现 | 兼容 |
|----|------|------|
| 编辑区 | 内联 `font-size` 动态计算 | - |
| 预览区/实页 | `paint-order: stroke fill; -webkit-text-stroke: 2px`（Chrome 120+） | Chrome < 120 → `[data-old-webkit]` → HarmonyOS Sans SC 粗体 + `-webkit-text-stroke:0.5px` |

> 🗣️ 白话：描述行就是板块里每一行文字（如"容量: 7/2"）。编辑区可以调文字颜色、透明度、加粗，还能加图片和数量角标。预览和实页会渲染出来，但没有颜色和透明度的编辑能力。×N 角标在老旧安卓上可能有兼容问题。

## 7. 图标组（icon_group）

图标组是 `info_card` 板块的一种描述类型，用于展示一组小图标（资源/物品），每组可配文字标题和服务器归属。

### 7.1 结构

| 元素 | 说明 |
|------|------|
| title | 可选标题（支持 `mdi:xxx` 前缀指定图标） |
| icons[] | 图标列表，每项含 `image_url`、`quantity`、`image_auto_color_mode`、`image_native_luminance` |
| server_states | 同描述行的三态归属（0/1/2），为空时用旧字段 `server` |

### 7.2 编辑区

- 图标组在板块编辑器内以行内缩略图展示（28×28px），每张图可点击唤出选图器
- 每张图右上角显示 `×N` 数量角标（`ic-qty`）
- 拖拽排序、`+` 按钮添加新图标
- 标题通过输入框编辑，不带 hr 分隔线
- server_states 通过 `descServerPopup`（同 §4.2）设置
- 无图片时显示虚线占位符 `+`

### 7.3 预览区/实页

**渲染模式由 `server_states` 决定，与有无 title/icons 无关**：

| 条件 | 渲染模式 | CSS 类 |
|------|---------|--------|
| `block_maps` 非空 | 色块行 | `ic-text ic-block-{sid}` |
| `linear_maps` 非空 | 线性行 | `ig-row-wrapper ic-linear-{sid}` |
| 无归属 | 普通行 | `ig-row-wrapper` |

> 唯一例外：无 title + 无图标 + 非 block 态 → 输出纯 `<hr>` 分隔线并跳过后续渲染（`continue`）。

**标题行**（有 title 时）：

| 模式 | 标题渲染 | 分隔 |
|------|---------|------|
| 线性/无归属 | `ig-title-badge`（小标签，0.65em） | `ig-title-line`（`<hr>` 填满剩余空间） |
| 色块态 | 纯文字（白色，继承 `.ic-text.ic-block-{sid}`） | 无 hr |

**图标行**：`ig-row` 容器内逐个渲染 `ig-item`，每项包含：
- 图片 `<img class="ig-img">`（28×28px，`border-radius:4px`）
- 可选 `ic-auto-dark`/`ic-auto-light`/`ic-auto-color` 反色 class
- ~~可选 `ic-qty` 数量角标~~（渲染代码预留但编辑器硬编码 `quantity:0`，实际永不显示）

**CSS 规则**：

| CSS 类 | 规则 |
|--------|------|
| `ig-title-badge` | `display:inline-flex;align-items:baseline;font-size:0.65em;padding:2px 6px` |
| `ic-linear-{sid} .ig-title-badge` | 服务器底色 + 服务器色文字（同 `ic-badge`） |
| `ig-item` | `position:relative;display:inline-flex;flex-shrink:0` |
| `ig-row` | `display:inline-flex;flex-wrap:wrap;gap:6px;align-items:center` |
| `ig-row-wrapper` | `flex-basis:100%;width:100%` |

The icon auto-color mode（`image_auto_color_mode`）应用 `ic-auto-dark`/`ic-auto-light` class，与描述行图片反色机制一致。

> 🗣️ 白话：图标组就是板块里的一排小图标（比如铁锭×50、木头×100），可以配个标题（如"建筑材料"），还能绑定某个服务器来着色。编辑器里拖拽排序，预览和实页里自动渲染成三种样式——纯分割线、彩色标签标题行、或整行染色的色块。

## 8. 组合图（Canvas 合成）

组合图是选图器中的特殊功能：选中多张图片后通过 Canvas 合成为一张 `data:image/png;base64` URL，用于板块标题图标、描述行图片或图标组内图标。

### 8.1 编辑区：合成流程

```
选图器 → 点击「组合」→ 勾选多张图片 → 点击「合并」
       → Canvas.drawImage() 逐张叠加
       → canvas.toDataURL('image/png') → data: URL
       → 调用对应 save 函数写入 editorData
```

**三个保存入口**（由 `composeIcons()` 分发）：

| 入口 | 目标 | 存储路径 | 反色 |
|------|------|----------|:--:|
| `saveComboIC` | 板块标题图标 | `content_blocks[bi].icon_url` | ✅ |
| `saveComboDesc` | 描述行图片 | `descriptions[di].image_url` | ✅ |
| `saveComboIG` | 图标组行内图标 | `descriptions[rowDi].icons[ii].image_url` | ✅ |

**反色处理**：
- 读取当前 mode（`off`/`normal`/`reverse`）
- mode=`normal` 时**异步**运行 `_detectComboBrightness(dataUrl, cb)` 检测合成图亮度
- 写入 `image_auto_color_mode` + `image_native_luminance`
- mode≠`normal` 时清空视觉 class（`ic-mode-normal`/`ic-mode-reverse` + filter）

**Canvas 合成逻辑**：
- 创建 `<canvas>` → 获取每张选中图片的 `<img>` → `drawImage` 逐层叠加 → `toDataURL`
- 结果以 Base64 内嵌（不上传服务器），直接存到 JSON 中

### 8.2 预览区/实页：渲染

组合图生成后存入 JSON 的是一个 `data:image/png;base64,...` 字符串，build_lovelace.py **不区分**组合图和普通图——统一走同一套图片渲染逻辑：

| 元素 | 编辑区（内存） | 预览区/实页 |
|------|:--:|------|
| 存储格式 | `data:image/png;base64,...` | 同（JSON 原样保留） |
| CSS 类 | 内联 filter + `ic-mode-*` | `ic-desc-img` / `ic-block-img` + `ic-auto-dark/light/color` |
| 反色渲染 | 编辑器内实时 class 更新 | `make_ic_css()` 根据 `image_native_luminance` 分配 class |
| 亮度检测 | JS `_detectComboBrightness`（异步） | Python 只读已存字段，不重新检测 |

### 8.3 样式差异

| 方面 | 编辑区 | 预览区/实页 |
|------|--------|----------|
| 合成预览 | Canvas 即时渲染（选图器内可见） | 单独一张图（与普通图无异） |
| ×N 角标 | 不显示（合成后 qty=0） | 同 |
| 反色 class | `.ic-mode-normal` / `.ic-mode-reverse` + filter | `ic-auto-dark` / `ic-auto-light`（由 lum 决定） |
| data: URL 体积 | Base64 大字符串（JSON 膨胀） | 同（嵌入 lovelace JSON） |

> ⚠️ **限制**：组合图用 `data:` URL 存储，不经过 HTTP。这意味着：① JSON 体积膨胀（一张组合图可能 10-100KB）；② HA 端无法重新下载；③ 编辑区删除原始 JSON 后组合图丢失。

> 🗣️ 白话：组合图就是在编辑器里把多张图片像拼贴画一样叠起来，生成一张新图。这个新图不是上传到服务器的，而是直接转成很长一串 Base64 编码文字塞进数据里。预览和实页看到的就是这张合成好的图，跟普通图片一样处理色彩和反色。

## 9. 默认值/提示

| 场景 | 编辑区默认 | 预览/实页默认 |
|------|----------|-------------|
| 板块标题 | placeholder="标题" | 无（显示文字） |
| 描述文字 | placeholder="描述文字" | 无（显示文字） |
| 颜色 | `auto` → `.auto-theme` outline | `auto` → 跟随 `--text` |
| 图片 | `+` 虚线框 placeholder | 无图不显示 |
| 图标 | `+` 虚线框 placeholder | 无图标不显示 |
| 数量 | `×N` 虚线框 placeholder | 0 不显示 |

> 🗣️ 白话：编辑区用虚线框和 placeholder 文字提示你可以填什么，保存后在预览和实页里这些提示就消失了。没填图片就不会显示图，没填数量就不会显示 ×N 角标。

## 10. 关键差异汇总

| 差异点 | 编辑区 | 预览/实页 | 影响 |
|--------|--------|----------|------|
| 描述行颜色/透明度 | ✅ (color dot + opacity dot) | ✅ (inline style / block bg) | 三层均支持颜色 |
| 颜色选择器 | color dot + opacity dot | 无独立 UI | 编辑区专属 |
| 图标组 | 拖拽排序 + 选图器 | 纯 CSS 渲染 + inline style | 编辑区有交互 UI |
| 组合图 | Canvas 合成 + `data:` URL 存储 | 同普通图（不区分来源） | data: URL 膨胀 JSON |
| 内联样式 | 大量 inline style | card_mod CSS | Shadow DOM 限制 |
| 主题切换 | JS toggle | URL param / HA 主题 | 实现路径不同 |

> 🗣️ 白话：总结一下，三层核心逻辑是一致的（颜色、反色、服务器标记都是同一套规则），差异主要在操作层面——编辑区有交互 UI 让你调参数，预览/实页只是把结果渲染出来。

---

> **结论**: 编辑区和预览/实页的核心着色逻辑一致（同源 `SERVER_MAP` + `make_ic_css`），差异主要在 UI 交互层（颜色选择器、亮度检测时序）和 CSS 注入方式（inline vs card_mod）。

### 11.5 🟡 P1：暗/亮主题变量统一

**现状**：预览区已自动统一——`preview_server.py` 直接读取 HA 主题 YAML，`--primary-text-color` 等变量自动可用。但**编辑区**自建了一套变量名（`--text`、`--bg`），与 HA 不互通，CSS 维护时需人工翻译。

**方案**：编辑区 CSS 追加别名映射：

```css
:root {
  --primary-text-color: var(--text);
  --primary-background-color: var(--bg);
}
```

**收益**：编辑区和实页可用同一套 CSS 选择器（如 `color: var(--primary-text-color)`），减少 card_mod 适配时的变量名转换。

### 11.6 🟢 P2：`[data-old-webkit]` 版本检测说明

**现状**：`navigator.userAgent.match(/Chrome\/(\d+)/) < 120` → 注 `data-old-webkit`。Chrome 120 是在 WebView 中实测 `paint-order` 渲染正确的首个版本。

**为何不用特征检测**：`@supports (paint-order: stroke fill)` 在 Chrome 119 也返回 `true`——语法支持但 WebView 渲染有 bug（基线偏移）。UA 版本精确对应了"bug 修好"的版本边界，特征检测无法区分"有能力"和"没 bug"。

**方案**：维持现状。定期复查 Chrome 市场份额，当 Chrome < 120 占比 < 1% 时直接删除此分支。

**收益**：无（说明项，不做改动）。

### 11.7 🟢 P2：组合图 `data:` URL 存储 — 暂不改动

**现状**：Canvas 合成后存为 `data:image/png;base64,...` 直接写入 JSON，一张图 10-100KB。

**方案**：暂不改动。理论上可存为服务端 PNG，但副作用（上传延迟、孤儿文件、旧数据兼容、并发冲突）超出当前收益。

**收益**：无（说明项，不做改动）。

### 11.8 🟡 P1：`ic-auto-color` 推定反色分支清理

**现状**：`lum` 缺失或为 `0.5` 时使用 `ic-auto-color`（`filter: var(--ic-icon-filter)`），不产生可见反色。静默吞掉异常，bug 难以发现。

**方案**：

| lum 值 | 行为 | 原因 |
|--------|------|------|
| `> 0.5` | `ic-auto-light` | 亮图 → 亮主题反色 |
| `≤ 0.5` 且 `≠ null` | `ic-auto-dark` | 暗图 → 暗主题反色（含 `0.5` 的中性亮度） |
| `null` | `ic-auto-dark` + **toast 报错** | 亮度检测失败，需修 bug |

1. 删除 `ic-auto-color` 类及所有相关 CSS
2. `lum == 0.5` 走 `ic-auto-dark`（中性亮度不反色也不报错）
3. `lum == null` 走 `ic-auto-dark` + toast 弹窗提示"亮度检测失败"
4. JS 端 `getImageLuminance` 回调中若 `lum` 为 null 同样 toast

**收益**：CSS 类从 3→2；`0.5` 正常处理不报警；`null` 有 toast 暴露 bug。
