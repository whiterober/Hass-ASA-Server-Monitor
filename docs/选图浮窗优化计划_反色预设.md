## 2. 新增数据文件：`icon_anti_color.json`

### 路径
`/config/www/asa-data/icon_anti_color.json`

### 结构
```json
{
  "categories": {
    "items": { "original": "light" },
    "armor": { "original": "dark" }
  },
  "files": {
    "/local/ASA/items/Thatch.png": { "original": "light" }
  }
}
```

> `original`: `"light"`=白底, `"dark"`=黑底。用户手动指定。仅此一个字段。

### 作用
1. **缩略图 filter**：`original` 有值时对缩略图应用 `filter: invert(1)` 展示反色预览效果
2. **圆点指示**：`original` 结合当前主题判断是否需要圆点及颜色

### 加载
`openIconPicker` 中 fetch（如未加载），存入 `window.__antiColorPresets`。

### 匹配规则
- `files[url]` 优先于 `categories[cat]`（文件级覆盖分类级）
- 两者均无匹配 → 不应用反色，无 filter，无圆点

## 3. 缩略图亮度指示圆点（白/黑，仅浮窗内）

### 渲染规则

直接读 JSON 的 `original` 字段（`"light"`=白底 / `"dark"`=黑底）。filter 反转后按主题判断：

| original | 当前主题 | filter 后 | 需要圆点？ | 圆点 |
|:--:|---------|-----------|:--:|------|
| dark | dark | 变亮 → 对比度够 | ❌ | — |
| dark | light | 变亮 → 与亮背景相近 | ✅ | 黑点白边 |
| light | dark | 变暗 → 与暗背景相近 | ✅ | 白点黑边 |
| light | light | 变暗 → 对比度够 | ❌ | — |

> 核心逻辑：filter 反转后缩略图和背景色相近 → 看不清 → 显示圆点提示。

### 渲染

```html
<div class="icon-picker-item" ...>
  <div class="icon-thumb-wrap">
    <img src="..." class="icon-picker-img" style="filter: invert(1)" ...>
    <span class="ac-dot ac-dark"></span>  <!-- 仅按需渲染 -->
  </div>
  <span>Riot Helmet.png</span>
</div>
```

### CSS（**样式统一写在 `make_ic_css()` 函数中，不散落在 HTML 内**）

`.ac-dot` / `.icon-thumb-wrap` 等指示器样式作为 ig-* 扩展规则追加到 build_lovelace.py 的 `make_ic_css()` 末尾，preview_server.py 通过 import 自动同步。ASA 后台 HTML 内不新增散落样式。

```css
.icon-picker-overlay .icon-thumb-wrap { position: relative; display: inline-block; line-height: 0; }
.icon-picker-overlay .ac-dot { position: absolute; top: 2px; right: 2px;
  width: 8px; height: 8px; border-radius: 50%; pointer-events: none; z-index: 1; }
.icon-picker-overlay .ac-dot.ac-light { background: #fff; border: 1px solid #333; }
.icon-picker-overlay .ac-dot.ac-dark  { background: #333; border: 1px solid #fff; }
```

## 4. 浮窗反色按钮 — 同步状态 + 延迟生效

### 初始状态同步

打开浮窗时读当前入口的 `image_auto_color_mode`（两态：`off` / `normal`）：
- `'off'` → 按钮灰色 "反色：关"
- `'normal'` → 按钮蓝色 "反色：开"

> `reverse` 不是独立按钮状态——它是 `image_auto_color_mode = 'reverse'` 时的亮度反转计算（`1 - lum`），按钮仍然显示"反色：开"（蓝色）。按钮始终只有关/开两态。

### 点击行为（仅切换状态，不立即执行）

点击按钮**只切换 UI 状态**（文字+颜色），不触发亮度检测、不写 `editorData`、不刷新 DOM。按钮记录一个 `_pendingAcMode` 表示用户意图。

### 关闭浮窗时判断

在 `closeIconPicker()`（关闭浮窗函数）中：

```javascript
function closeIconPicker() {
  // 仅在以下条件全部满足时才走反色流程：
  // 1. 没选新图（选图流程已自行处理反色）
  // 2. 入口图标存在（有图可反色）
  // 3. 反色状态确实变了
  if (!_pickerSelectedNew 
      && hasCurrentEntryImage(ctx) 
      && _pendingAcMode !== _originalAcMode) {
    applyAntiColorToCurrentEntry(ctx, _pendingAcMode);
  }
  // ... 移除 overlay ...
}
```

### 不影响现有选图+反色流程

当用户在浮窗里先切反色按钮再选图时：
1. `_pendingAcMode` 已更新
2. `selectDescImage` 被调用 → `_pickerSelectedNew = true`
3. `closeIconPicker` 中 `!_pickerSelectedNew` 为 false → 跳过延迟生效
4. `selectDescImage` 内部已有完整的反色处理逻辑

> **无入口图标保护**：`hasCurrentEntryImage(ctx)` 检查当前入口是否真的有图。空图标时按钮可点但不生效。

## 5. 有入口图标时最近第一张预填

### 触发条件
**入口图标已存在**时（slot 不为空），打开浮窗即用最近第一张预填入口图标位。入口图标为空时，最近分类正常显示 20 个图标，不做预填。

### 最近列表后移逻辑

`recent.shift()` 取走第一张填入入口图标，后续项前移补位（原 recent[1]→recent[0]，…），末位挤出保持总数 ≤20。

```javascript
function prefillFromRecent(ctx) {
  if (!hasCurrentEntryImage(ctx)) return;  // 无入口图标 → 不预填
  var recent = JSON.parse(localStorage.getItem('asa-recent-icons') || '[]');
  if (!recent.length) return;
  var first = recent.shift();
  localStorage.setItem('asa-recent-icons', JSON.stringify(recent));
  refreshRecentTab();
  selectDescImage(first.url, first.name, ctx);
}
```

```
有入口图标：最近=[A,B,C,…] → A→入口图标, 最近=[B,C,…], 总数-1
无入口图标：最近正常显示20个，不触发预填
```
