### 步骤 3：升级 `selectIcon(item, ctx)` — 全模式转发（不重写旧逻辑）

**`selectIcon` 本身只做路由转发**，不包含任何选择逻辑：

```javascript
function selectIcon(item, ctx) {
  var url = item.getAttribute('data-url') || '';
  if (!url) return;
  switch(ctx.mode) {
    case 'info_card_title':
      selectInfoCardIcon(item, ctx.bi);     // 现有函数，不动
      break;
    case 'desc_row':
    case 'icon_group':
      selectDescImage(item, ctx.bi, ctx.ii !== undefined ? ctx.ii : ctx.di); // 现有函数，不动
      // icon_group 的 _igRowDi 传递由 openIconPicker 在创建 overlay 时设置
      break;
    case 'base_row':
    case 'base_item':
      selectBaseIcon(item, ctx);            // 新增函数
      break;
  }
}
```

**新增 `selectBaseIcon(item, ctx)`**：
- 5 步流程参考 `selectDescImage` 的已验证模式（L7874 起）
- 操作 `baseData` 而非 `editorData`
- 最后调用 `renderBaseRef()` 而非 `reRender()`

**改动量**：~50 行（仅新增 selectBaseIcon，selectInfoCardIcon/selectDescImage 不动）

## 🛡️ 回归保护 — 零破坏原则

> **核心铁律**：`selectInfoCardIcon`(L8213) 和 `selectDescImage`(L7874) 是经历过 v1064-v1095 共 ~30 个版本反复修复的战斗验证代码。**这两个函数本身不做任何修改。**

### 保护策略：旧 handler 不动，新 `selectIcon` 只做转发

```
新 openIconPicker(ctx)  → 统一生成 picker HTML（onclick 调用 selectIcon）
                                   │
新 selectIcon(item, ctx) ──────────┤
  │                                │
  ├─ mode=info_card_title ─→ selectInfoCardIcon(item, ctx.bi)  ← 现有函数，不动
  ├─ mode=desc_row       ─→ selectDescImage(item, ctx.bi, ctx.di) ← 现有函数，不动
  ├─ mode=icon_group     ─→ selectDescImage(item, ctx.bi, ctx.ii) ← 现有函数，不动
  │                          + overlay._igRowDi 传递（现有逻辑）
  ├─ mode=base_row       ─→ selectBaseIcon(item, ctx)   ← 新增，参考 selectDescImage 模式
  └─ mode=base_item      ─→ selectBaseIcon(item, ctx)   ← 新增，同上
```

> `selectInfoCardIcon` / `selectDescImage` 的调用签名和内部逻辑 **一行都不改**。
> `selectIcon` 仅做一个 `switch(ctx.mode)` 转发——相当于把旧版 `openIconPicker(ctx)` 的路由逻辑移到 `selectIcon` 里。

### 新增函数隔离

`selectBaseIcon(item, ctx)` 是本次唯一新增的选择处理函数。它参考 `selectDescImage` 的 5 步流程但操作 `baseData` 而非 `editorData`：

```javascript
function selectBaseIcon(item, ctx) {
  // 1. 写入 URL → brSRIcon / brItemIcon 隐藏域
  // 2. 读颜色模式按钮 → image_auto_color_mode
  // 3. 写入 baseData（不用 try/catch 静默吞错）
  // 4. brightness 检测 → image_native_luminance + CSS class + 蓝点
  // 5. renderBaseRef() 重渲染
}
```

> 此函数独立存在，不触碰任何现有选择函数。即使它有 bug，也仅影响基地编辑器，不影响服务器规则和部落运维。
