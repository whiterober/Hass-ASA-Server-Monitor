# 部落基地编辑器重构 + 三编辑器代码统一计划

> 版本: v1.0 | 日期: 2026-07-11 | 基线: `_pre_tribe_base`

---

## 一、现状分析

### 1.1 三编辑器对比

| 维度 | 服务器规则 | 部落运维 | 部落基地 |
|---|---|---|---|
| JSON 文件 | `server_rules.json` | `tribe_ops.json` | `asa_base_quick_ref.json` |
| 根结构 | `{tabs:[...]}` | `{tabs:[...]}` | `{servers:{...}}` |
| 嵌套层级 | tabs→blocks | tabs→blocks | servers→tabs→**sections**→blocks→**rows** |
| block 类型 | info_card, divider, text, table, map_filter, supply_card 等 9 种 | 同左 | base_storage, base_device, base_creature, base_badge 等 6 种（**完全不同**） |
| 编辑器函数 | `renderMixedEditor` → `renderMixedContent` | `renderTribeOps` → `renderMixedContent` | `renderBaseRef`（独立实现） |
| 保存函数 | `saveMixedEditor` → `saveMCBlocks` | `saveTOTab` → `saveMCBlocks` | `saveBaseTab`（独立实现） |
| lovelace 渲染 | `render_server_rules` → `render_tab_html` | `render_tab_html` (mixed) | `build_section_html` + `make_raw_card` |

### 1.2 核心问题

1. **部落基地数据结构完全不同** — 多了 `sections` 中间层 + 6 种特有 block 类型
2. **渲染输出不一致** — base 用 `make_raw_card` 生成旧式 HTML，mixed 用 card 式 `info_card` 块
3. **编辑器代码完全独立** — `renderBaseRef` / `saveBaseTab` 与 mixed 编辑器零代码共享
4. **服务器规则和部落运维**已几乎共用代码（仅文件名不同），部落基地是孤岛

## 三、Phase 2：部落基地编辑器结构对齐

### 3.1 目标

将部落基地编辑器改造为与服务器规则/部落运维一致的结构：
- 数据格式统一为 `tabs → content_blocks`（含 `info_card`、`icon_group`、`divider` 等）
- 编辑器 UI 统一（卡片式编辑）
- 预览/实页渲染统一（`render_tab_html`）
- **保留地图栏**（server 选择器，作为 base 独有的筛选功能）

### 3.2 数据迁移方案

#### 旧结构 → 新结构映射

```
servers[serverId]
  └── tabs[ti]
        ├── name → tab.name
        ├── description → tab.description
        └── sections[si]
              ├── name → divider block (title=section.name)
              └── content_blocks[bi]
                    ├── base_storage → info_card (标题=设备名, 描述=容量+分类项)
                    ├── base_device → info_card (标题=设备名, 描述=属性)
                    ├── base_creature → info_card (标题=生物名, 描述=属性)
                    ├── base_badge → info_card (标题=徽章名, 描述=属性)
                    └── ...
```

#### 关键数据映射

| 旧字段 | 新字段 | 说明 |
|--------|--------|------|
| `row.device_icon_url` | `info_card.images[0].image_url` | 设备图标 |
| `row.capacity_main` / `capacity_sub` | `info_card.descriptions[].text` (bold) | 容量显示为粗体描述行 |
| `row.actions[]` (power/water/fire/tek) | `info_card.descriptions[]` 中带图标 | 行动图标转为 icon_group |
| `row.categories[].name` | `info_card.descriptions[]` 中带 `[badge]` | 分类名用徽章样式 |
| `row.categories[].items[]` | `info_card.descriptions[]` 中带图片 | 物品渲染为图标行 |
| `section.name` | `divider` block (title=name) | 分区标题转为分隔线 |
| `server.name` (地图) | 地图筛选 `map_filter` + `highlight` | 每个 info_card 带 server 属性 |

### 3.3 编辑器 UI 改造

| 旧 UI | 新 UI | 说明 |
|--------|--------|------|
| `renderBaseRef()` 独立渲染 | 复用 `renderMixedContent()` | 去掉 sections 面板、容量字段等 |
| 6 种 base_* block 按钮 | 改为 `+ 卡片` / `+ 分隔线` / `+ 地图筛选` | 与 mixed 一致 |
| 服务器 TAB 切换 | 保留，作为 base 独有功能 | 通过 `map_filter` + `highlight` 实现跨服 |
| sections 折叠面板 | 去除，改为 divider 分隔 | sections → tab 级别内 divider |
| 存储行头/分类卡片 | 去除，改为 info_card 内部的 icon_group | 物品用图标组渲染 |
| 行动图标选择器 | 保留在 info_card 描述行内 | 作为 icon_group 的图标 |
| 容量字段 (主/副) | 改为 info_card 描述行的粗体文字 | `**主容量** / **副容量**` |

### 3.4 渲染改造

**build_lovelace.py 改动**：
- 基地视图从 `build_section_html() + make_raw_card()` 改为走 `render_tab_html()`
- 11 个 `base_*` 视图全部改为 `mixed_content` 类型渲染
- 添加 server-level 的 `map_filter` 支持（按服务器筛选卡片）

**数据文件改动**：
- `asa_base_quick_ref.json` 结构从 `{servers: {}}` 改为按服务器拆分为 11 个 JSON（或保持单文件但路径扁平化）
- 每个服务器作为一个独立的 `tabs` 数组

## 四、Phase 3：三编辑器代码统一源

### 4.1 可行性评估

**结论：可行，但需要引入"编辑器模式"参数。**

服务器规则和部落运维几乎已完全共用代码（仅文件名不同）。Phase 1 完成后，部落基地也将使用相同的 `renderMixedContent` / `saveMCBlocks`。此时三编辑器唯一差异为：

| 差异点 | 统一方案 |
|--------|---------|
| JSON 文件名 | 参数化：`config = {file:'server_rules.json', label:'服务器规则'}` |
| 数据变量名 | 统一为 `editorData`，不再分 `serverRulesData` / `tribeOpsData` / `baseData` |
| 全局状态 | `editorSelectedTab` 统一 |
| Base 的 server 选择器 | 条件渲染：`if (editorMode === 'base')` 显示地图栏 |
| Base 的跨服 TAB 同步 | Base 模式专属逻辑 |
| 导航按钮文案 | 从 config 读取 |

### 4.2 统一架构

```javascript
// 编辑器配置（替代分散的全局变量）
const EDITOR_CONFIGS = {
  server_rules: { file: 'server_rules.json', label: '服务器规则', hasMapBar: false },
  tribe_ops:    { file: 'tribe_ops.json',    label: '部落运维',   hasMapBar: false },
  base_ref:     { file: 'base_quick_ref.json', label: '部落基地', hasMapBar: true  },
};

// 当前模式
let editorMode = 'server_rules';  // 'server_rules' | 'tribe_ops' | 'base_ref'
let editorData = null;            // 统一数据对象
let editorSelectedTab = null;     // 统一当前 TAB

// 统一渲染入口
function renderEditor() {
  const cfg = EDITOR_CONFIGS[editorMode];
  if (cfg.hasMapBar) renderServerBar();  // base 独有：地图选择栏
  renderMixedContent(editorSelectedTab); // 共用
}

// 统一保存入口
function saveEditor() {
  const cfg = EDITOR_CONFIGS[editorMode];
  saveTabToMemory(editorSelectedTab);
  saveMCBlocks(editorSelectedTab);
  saveJSON(cfg.file, editorData);
}
```

### 4.3 可删除的重复函数（Phase 2 完成后）

| 函数 | 替代 |
|------|------|
| `loadServerRules()` / `loadTribeOps()` / `loadBaseRef()` | 统一为 `loadEditorData(mode)` |
| `renderMixedEditor()` / `renderTribeOps()` / `renderBaseRef()` | 统一为 `renderEditor()` |
| `saveMixedEditor()` / `saveTOTab()` / `saveBaseTab()` | 统一为 `saveEditor()` |
| `addEditorTab()` / `addTOTab()` / `addBRTab()` | 统一为 `addTab()` |
| `_delSRTab()` / `_delTOTab()` / `rmBRTab()` | 统一为 `rmTab()` |
| `addMCBlock()` / `addBRSecBlock()` | 统一为 `addBlock()` |

### 4.4 关键挑战

1. **Base 的 sections 中间层** — Phase 1 已解决（section → divider block）
2. **Base 的服务器切换** — 保留为条件功能，不纳入通用代码
3. **Base 的跨服 TAB 同步**（一个 TAB 改了，其他服务器的同名 TAB 也改）— Base 专属
4. **Base 的复制功能** (`copyBRFrom`) — 保留为 Base 模式专属
5. **旧数据兼容** — 旧格式 `raw_html` + `body` HTML 需要迁移脚本一次性转换
