# 计划：服务器选择器新增「默认/自定义颜色」选项

## 目标

为所有行级元素（描述行、图标组）的服务器选择弹窗新增一个「默认」选项 (`_default`)，替代当前无选中时的默认地图图标。该选项使用用户自己选的颜色，而非预设地图色。

## 核心逻辑

| 场景 | 行为 |
|------|------|
| 新建描述行 | 自动选中 `_default` 线性态（`server_states: {_default: 1}`） |
| 线性态 (`_default: 1`) | 不设额外颜色，直接使用用户选的颜色（等效当前默认效果） |
| 色块态 (`_default: 2`) | 底色使用用户当前行级选的颜色（`dcolor`） |
| 切换地图 | 自动修改当前行颜色为地图色（同现有逻辑） |
| 切换默认 | **不修改**当前行颜色，保持用户自选色 |

## 数据结构

`FIXED_STYLES` 新增：
```javascript
_default: {c:'auto', n:'palette', label:'默认'}
```

插入顺序：`_default` → `_hint` → `_warn` → `_remark`

## 改动清单

### 1. asa-admin.html

#### 1a. FIXED_STYLES 定义
- 新增 `_default: {c:'auto', n:'palette', label:'默认'}`
- 放在 `_hint` 之前（最前面）

#### 1b. getServerBadge 函数
- 处理 `s.c === 'auto'` 的情况：返回调色板图标，颜色用 `var(--text)`

#### 1c. selectDescServer 函数
- 自动设颜色逻辑：`if (s && s.c && s.c !== 'auto')` 才设颜色
- `_default` 的 `c: 'auto'` 跳过颜色覆盖

#### 1d. 新建描述行默认值
- `addICDesc`: `server_states` 默认 `{_default: 1}`（replace `{}`）
- `addICIconGroup`: 同上
- `addICCopyKey`: 无 server 按钮，不需要

#### 1e. toggleDescServerPopup
- `_default` 在固定样式区最前面显示（在 `_hint` 之前）
- `Object.keys(FIXED_STYLES)` 保持插入顺序

### 2. build_lovelace.py

#### 2a. FIXED_STYLES 同步
```python
'_default':{'label':'默认','icon':'mdi:palette','color':'auto'},
```

#### 2b. 线性态渲染
- `_default` 线性态：不设 per-map 颜色，走 `dcolor`（用户自选）
```python
if _sid == '_default':
    # 不设额外颜色，使用 dcolor
    pass
```

#### 2c. 色块态渲染
- `_default` 色块态：inline `background: dcolor`
```python
if _sid == '_default' and block_maps:
    dstyle += 'background:{};'.format(dcolor) if dcolor else ''
```

#### 2d. CSS 规则
- `.ic-block-_default`：不设固定背景色（由 inline style 控制）

### 3. preview_server.py
- FIXED_STYLES 同步
- CSS 规则同步

## 实施步骤

1. 两文件 FIXED_STYLES 加 `_default`
2. `getServerBadge` 处理 `auto` 色
3. `selectDescServer` 加 `c !== 'auto'` 判断
4. 新建描述行默认 `{_default: 1}`
5. **旧数据迁移**：三个 JSON 数据文件中 `server_states: {}` 的行 → 写入 `{_default: 1}`
6. `build_lovelace.py` 渲染：`_default` 线性态跳过，色块态 inline background
7. 部署 + 保存Tab

## 兼容性

- 旧数据中 `server_states: {}` 的描述行保持原样（无过滤，始终可见）
- 旧数据中 `server_states: {Isl: 1}` 等不受影响
- 只有新创建的描述行自动带 `_default: 1`

## 旧数据迁移：无选中 → 直接写入 _default 线性态

> 当前已有描述行如果「一个都没选」（`server_states: {}` 或字段缺失），**直接改数据写入 `_default: 1`**，不做兼容层。
