## 二、任务 1：最近选图

### 2.1 设计方案

**存储**：`localStorage` 键 `asa-recent-icons`，值为 JSON 数组 `[{url, name, cat}]`，最多 20 条，新选图插入头部，重复则移到头部。

**展示**：在分类标签栏最前面加「🕐 最近」Tab，点击显示最近 20 个图标。

**性能**：
- localStorage 读写纯 JSON，零网络请求
- 最近 20 个图标渲染量极小（vs 3000+ 全量）
- 无额外文件依赖

### 2.2 改动点（~30 行）

#### (a) 新增加/读取函数

```javascript
var RECENT_KEY = 'asa-recent-icons';
var RECENT_MAX = 20;

function addRecentIcon(url, name, cat) {
  var recent = [];
  try { recent = JSON.parse(localStorage.getItem(RECENT_KEY) || '[]'); } catch(e) {}
  recent = recent.filter(function(r) { return r.url !== url; });
  recent.unshift({ url: url, name: name, cat: cat });
  if (recent.length > RECENT_MAX) recent.length = RECENT_MAX;
  localStorage.setItem(RECENT_KEY, JSON.stringify(recent));
}

function getRecentIcons() {
  try { return JSON.parse(localStorage.getItem(RECENT_KEY) || '[]'); } catch(e) { return []; }
}
```

#### (b) 在 `selectBaseIcon` / `selectDescImage` / `selectInfoCardIcon` 中调用

在每个选图成功回调末尾加一行：
```javascript
addRecentIcon(url, itemName, catName);
```

实际插入位置：当前选图函数确定 URL 后，从 DOM 取 `item.getAttribute('data-url')` 和对应的 name/cat。

> ⚠️ 但现有的 `openIconPicker` 渲染时已经知道 name，可以直接在图标 `onclick` 时记录。

**更轻量方案**：不修改各个 select 函数，直接在 `openIconPicker` 的图标渲染循环中，给每个 `.icon-picker-item` 的 onclick 外层包一个记录函数：

```javascript
// 原有: onclick='if(comboMode){...}else{selectDescImage(this,...)}'
// 改为: onclick='if(comboMode){...}else{addRecentIcon(this.dataset.url,this.querySelector("span").textContent,this.dataset.cat);selectDescImage(this,...)}'
```

#### (c) 在 `openIconPicker` 分类栏加「🕐 最近」Tab

```javascript
// 在 catOrder.forEach 之前插入
html += "<span class='server-tab icon-cat-tab' data-cat='__recent__' onclick='filterIconPickerCat(this)'>🕐 最近<span class='cat-count-badge'>"+getRecentIcons().length+"</span></span>";
```

#### (d) 在图标网格中渲染最近图标（选中「最近」Tab 时）

修改 `filterIconPicker` 或直接在渲染循环中处理：
- 当 `iconPickerCat === '__recent__'` 时，不从 `list` 渲染，改为从 `getRecentIcons()` 渲染

```javascript
// 在 openIconPicker 的 list.forEach 之前，先检查是否最近 Tab
// 如果 catOrder 包含 __recent__ 且当前选中，则用 getRecentIcons() 替代 list
```

## 三、任务 2：中文搜索

### 3.1 设计方案

**核心思路**：本地预构建 `{英文名小写 → 中文名}` 映射表，仅上传 ~1MB 的结果文件到 HA。

**数据来源（均在本地）**：
- `汉化/ShooterGame.po`（英文模板，2.5MB）：`msgctxt "Content,NUMERIC_ID"` → `msgid "English Name"`
- `A:\...\zh\ShooterGame.json`（中文本地化）：`{Category: {NUMERIC_ID: "中文名"}}`

**构建流程**（本地一次性执行）：
1. 解析 PO → `{NUMERIC_ID: englishText}`（19,340 条）
2. 解析 zh JSON → `{NUMERIC_ID: chineseText}`（28,142 条）
3. 交叉匹配 → `{englishTextLower: chineseText}`（14,576 公共 ID）
4. 过滤：排除 >80 字符、含换行、含句号空格、小写开头的句子（保留纯名称）
5. 输出 → `tmp/icon_zh_map.json`（~1.1MB，20,672 条映射）→ **仅此文件上传 HA**

### 3.2 本地构建脚本

**文件**：`汉化/_build_icon_zh_map.py`，可重复运行。

```bash
python 汉化/_build_icon_zh_map.py
# 输出: 汉化/icon_zh_map.json (~1.1MB)
```

上传仅需：
```bash
sftp.put('汉化/icon_zh_map.json', '/config/www/asa-data/icon_zh_map.json')
```

> PO 文件 2.5MB 和 zh JSON 始终保留在本地开发环境，不占用 HA 服务器空间。

### 3.4 前端集成

#### (a) 加载映射表

在 `loadIconList` 中并行加载：
```javascript
var iconZhMap = {};
function loadIconZhMap(cb) {
  if (Object.keys(iconZhMap).length) { cb(iconZhMap); return; }
  fetch('/local/asa-data/icon_zh_map.json?t=' + Date.now())
    .then(function(r) { return r.json(); })
    .then(function(data) { iconZhMap = data; cb(data); })
    .catch(function() { cb({}); });
}
```

#### (b) 修改 `filterIconPicker`

```javascript
function filterIconPicker() {
  var q = (document.getElementById("iconPickerSearch")?.value || "").toLowerCase();
  document.querySelectorAll(".icon-picker-item").forEach(function(el) {
    var name = (el.getAttribute("data-name") || "").toLowerCase();
    var cnName = (iconZhMap[name] || "").toLowerCase();
    var matchSearch = !q || name.includes(q) || cnName.includes(q);
    var matchCat = !iconPickerCat || el.dataset.cat === iconPickerCat;
    el.style.display = (matchSearch && matchCat) ? "" : "none";
  });
}
```

> ⚠️ 需要在渲染时为每个 `.icon-picker-item` 添加 `data-name` 属性

```javascript
// 渲染时加 data-name
html += "<div class='icon-picker-item' ... data-name='" + item.name + "' ...>";
```

### 3.5 估算开销

| 指标 | 值 |
|------|-----|
| `icon_zh_map.json` 大小 | ~1.1MB（20,672 条中文映射） |
| PO + zh JSON（本地） | 不上传，仅在构建时使用 |
| 前端内存 | 同大小，加载后缓存全局变量 |
| 搜索时 | Object 键查找 O(1)，字符串 `includes` O(n) |
| 网络请求 | 1 次（首次打开选图浮窗时懒加载） |

### 3.6 性能优化要点

- **本地构建**：PO 2.5MB + zh JSON 不占用 HA 服务器空间
- **过滤长文本**：排除 >80 字符 / 含换行 / 句号 / 小写开头的描述性文本
- **懒加载**：只在 `openIconPicker` 首次调用时加载映射表
- **紧凑 JSON**：`separators=(',',':')` 去掉空格
- **双向映射**：同时注册 `chitin_boots` 和 `chitin boots` → 中文

## 六、风险与注意事项

- PO 文件 2.5MB，解析需 1-2 秒，仅服务器端一次性执行，不影响前端
- 部分图标名在 PO 中可能无对应条目（如社区图标），此时中文搜索退化为英文搜索
- `localStorage` 在隐私模式下可能不可用，需 try/catch 包裹
- 中英文映射建立在下划线/空格互转基础上，边缘情况（如 `X-Species`）需验证
