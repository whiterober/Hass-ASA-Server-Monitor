# 🔍 一键搜索扩展方案 — 预留槽位

> 📅 2026-07-09 | 50 个槽位一次性建好，新增按钮只填不建

---

## ⚠️ 前置铁律

- **改 YAML 前**：必须先 SSH SFTP 拉取服务器最新 `configuration.yaml` / `scripts.yaml` 覆盖本地，禁止假设本地最新。
- **改 lovelace 前**：必须先拉取 `/config/.storage/lovelace.lovelace` 基线（即 `/homeassistant/.storage/lovelace.lovelace`）。
- **上传后**：必须执行 `python3 /config/build_lovelace.py` 重建 ASA 视图。

> 详见 `HASS切屏发送_落地实施方案.md` 铁律 1~3。

---

## 🎯 目标

一次部署 50 个预留槽位。以后新增按钮只需在 HA 界面改 `input_text` 的值，**不碰 YAML、不重启**。

---

## 原理

```
一次性建好 50 组 input_text + 50 个 search_dyn 脚本（初始值空）
        ↓
需要新按钮 → 打开对应 input_text → 填标签+值 → 刷新
        ↓
auto-entities 自动展示所有非空按钮 → 点按钮 → MQTT → 游戏
```

---

## 一次性部署

以下 50 组全部建好，初始为空，按钮不显示。需要时填值即出现。

### `configuration.yaml`

```yaml
input_text:
{% for N in range(1, 51) %}
  sb_{{ N }}_label:
    name: "按钮{{ N }}-标签"
    initial: ""
    max: 50
  sb_{{ N }}_value:
    name: "按钮{{ N }}-值"
    initial: ""
    max: 50
{% endfor %}
```

### `scripts.yaml`

```yaml
{% for N in range(1, 51) %}
search_dyn_{{ N }}:
  alias: "{{ '{{ states(\"input_text.sb_{{ N }}_label\") }}' }}"
  sequence:
    - service: mqtt.publish
      data:
        topic: "homeassistant/button/GAME/SendText/action"
        payload: "{{ '{{ states(\"input_text.sb_{{ N }}_value\") }}' }}"
{% endfor %}
```

> 🔧 实际执行时用 Python 批量生成，不用手写。

---

## 日常使用

| 操作 | 怎么做 |
|------|--------|
| ➕ 新增 | 找到空槽位 → HA 界面改 `sb_N_label` + `sb_N_value` → 刷新 |
| ✏️ 修改 | 改对应 `input_text` 的值 |
| ❌ 删除 | 清空 label 和 value |

---

## Lovelace（方舟视图）

```json
{
  "type": "custom:auto-entities",
  "card": {
    "type": "entities",
    "title": "⚡ 一键搜索"
  },
  "filter": {
    "include": [
      { "entity_id": "script.search_dyn_*" }
    ],
    "exclude": [
      { "state": "" }
    ]
  },
  "sort": { "method": "entity_id" }
}
```

> 空按钮（label 为空）被 auto-entities 自动隐藏。

## 涉及文件

| 文件 | 操作 |
|------|------|
| `configuration.yaml` | 一次性追加 100 个 input_text |
| `scripts.yaml` | 一次性追加 50 个 search_dyn_N |
| `lovelace`（方舟） | 替换 ⚡一键搜索 为 auto-entities |
