# 计划：_default 色块态修复 + 四种浮窗样式统一

## 一、问题 1 修复：渲染模板中 `_default` 色块态按钮图标消失

### 根因

`renderMixedEditor` / `reRender` 中描述行的服务器按钮渲染模板（L2719-L2752）直接使用 `sm.c` 和 `var(--primary-background-color)`，对 `_default`（`c: 'auto'`）无回退：

```javascript
// L2719 附近 — 按钮样式
if (ast === 2) return '...background:' + sm.c + ';border:none';
// _default → background:auto ❌

// L2752 附近 — 图标 HTML
if (ast === 2) return '<i ... style="color:var(--primary-background-color)">';
// ASA 后台不定义此变量 → 图标不可见 ❌
```

### 方案

在渲染模板中增加 `_default` 回退逻辑，与 `updateDescSrvBadge` 保持一致：

```javascript
// 判断是否为 _default
var isDefault = (ak === '_default');

// 按钮样式
if (ast === 2) {
  var bg = isDefault ? '#666' : sm.c;
  return '...background:' + bg + ';border:none';
}

// 图标 HTML
if (ast === 2) {
  var iconColor = isDefault ? '#fff' : 'var(--primary-background-color)';
  return '<i ... style="color:' + iconColor + '">';
}
```

### 影响范围

- `asa-admin.html`：2 处渲染模板（描述行 + 图标组，L2719 和 L2752 附近）
- 需确认 `addICDesc` / `addICIconGroup` 初始 HTML 中的 `getServerBadge('')` 是否也需要同步（当前无问题，因为新建时 state 为 linear，linear 态已处理）

## 二、问题 2 修复：`ic-block-_default` CSS 完整生成

### 根因

`build_lovelace.py` 中 `_default` 被完全跳过 CSS 生成（`if fv.get('color') == 'auto': continue`），导致 `ic-block-_default` 无 `border-radius`、`padding`、`color` 等样式。

### 方案

**不为 `_default` 跳过全部 CSS**，而是为它生成完整的块规则，仅把颜色部分用回退值 `#666`：

```python
for fk, fv in FIXED_STYLES_MAP.items():
    fc = fv.get('color', '#666')
    if fc == 'auto':
        fc = '#666'  # _default fallback
    # 正常生成完整 CSS
    IC_CSS += 'ha-card .ic-linear-{} .mdi,ha-card .ic-linear-{} ha-icon{{color:{}!important}}'.format(fk, fk, fc)
    IC_CSS += 'ha-card .ic-text.ic-block-{}{{background:{}!important;border-radius:6px!important;padding:2px 6px!important;color:var(--primary-background-color)!important}}'.format(fk, fc)
    IC_CSS += 'ha-card .ic-text.ic-block-{} .mdi,ha-card .ic-text.ic-block-{} ha-icon{{color:var(--primary-background-color)!important}}'.format(fk, fk)
    r = int(fc[1:3], 16); g = int(fc[3:5], 16); b = int(fc[5:7], 16)
    IC_CSS += 'ha-card .info-card-block.ic-block-{}{{background:rgba({},{},{},0.08)!important}}'.format(fk, r, g, b)
```

> ⚠️ 六处 FIXED_STYLES_MAP 的循环全部改为不跳过 `_default`，仅对 `color=='auto'` 做回退。

### 同时移除 inline background

`build_lovelace.py` 描述行渲染中的 `inline background` 补丁（之前加的）：

```python
# 删除这段
if '_default' in block_maps:
    dstyle += 'background:{};'.format(dcolor or 'var(--primary-background-color)')
```

CSS 规则已提供 `background`，不需要 inline 覆盖。

### 影响范围

- `build_lovelace.py`：6 处 FIXED_STYLES_MAP 循环 + 1 处 inline background 删除
- `preview_server.py`：3 处 FIXED_STYLES_MAP 循环（同步修改）

## 四、四种浮窗样式统一方案

### 统一目标

| 属性 | 统一值 | 来源 |
|------|--------|------|
| 背景 | `var(--card)` | 已有，一致 |
| 边框 | `1px solid var(--border)` | 已有，一致 |
| 圆角 | `8px` | 统一到 8px（颜色/透明度当前为 6px） |
| 内边距 | `6px` | 统一到 6px（颜色/透明度当前 `4px 6px`） |
| 阴影 | `0 4px 16px rgba(0,0,0,0.45)` | 统一到 0.45（服务器当前 0.15） |
| 遮罩层 | ❌ 无 | 数量弹窗去掉遮罩层 |
| z-index | `9999` | 统一（颜色/透明度当前 99999） |
| 显隐方式 | CSS class `.show` | 数量/服务器改为 class 显隐 |
| 关闭方式 | 捕获阶段 `document.click` | 已有，一致 |

### CSS 统一

新增一个共享 CSS 类，四种弹窗共用：

```css
.asa-popup {
  display: none;
  position: fixed;
  background: var(--card);
  border: 1px solid var(--border);
  border-radius: 8px;
  padding: 6px;
  z-index: 9999;
  box-shadow: 0 4px 16px rgba(0,0,0,0.45);
}
.asa-popup.show {
  display: flex;
}
```

### 各弹窗改造

#### 颜色弹窗（当前用 `.cat-color-popup`）

- 类名改为 `cat-color-popup asa-popup`
- `<style>` 中删除重复的背景/边框/圆角等声明，只保留颜色弹窗特有样式（`.cat-color-opt` 等）

#### 透明度弹窗（当前用 `.cat-opacity-popup`）

- 类名改为 `cat-opacity-popup asa-popup`
- 同上，只保留特有样式

#### 数量弹窗（当前 `toggleQtyPopup` 创建/销毁）

- 改为 class 显隐模式
- 弹窗元素在页面初始化时创建，后续只 toggle `show` class
- 去掉遮罩层
- toggle 检测统一用 `dataset` 属性

#### 服务器弹窗（当前 `toggleDescServerPopup` 创建/销毁）

- 改为 class 显隐模式
- 弹窗内容动态更新（每次打开时更新按钮状态）
- 去掉 `_descPopupOpen` 标记，改用 `dataset.triggerId` 检测

### 关闭逻辑统一

捕获阶段 handler 已有，只需确保所有弹窗都用 `.show` class 控制显隐：

```javascript
document.addEventListener('click', function(e) {
  if (e.target.closest('.asa-popup')) return;  // 点在弹窗内
  if (e.target.closest('.cat-color-dot, .cat-opacity-dot, .btn-server, .qty-btn')) return;  // 触发器
  document.querySelectorAll('.asa-popup.show').forEach(p => p.classList.remove('show'));
}, true);
```

### 影响范围

- `asa-admin.html`：
  - `<style>` 中新增 `.asa-popup` 类，精简 `.cat-color-popup` / `.cat-opacity-popup` / `.qty-popup-overlay` / `.qty-popup-panel` 的重复属性
  - 重写 `toggleQtyPopup`（改为 class 显隐）
  - 重写 `toggleDescServerPopup`（改为 class 显隐）
  - 修改 `closeAllPopups`（移除 `.show` 而非删除元素）
  - 修改捕获阶段 handler
