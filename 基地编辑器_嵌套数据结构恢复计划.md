## 一、现状

### 数据格式
```json
{ "tabs": [...] }  // flat，所有服务器数据混在一起
```

### 编辑器行为
- 切换地图（`brSelectedServer`）→ `reRender()` → `renderMixedContent(tab)`
- `renderMixedContent` **不区分服务器**，渲染全部 tabs
- 所有服务器显示相同的 tabs 列表

### build_lovelace 兼容
```python
# L2712：flat 格式 fallback
if not bq_servers and bq_data.get('tabs'):
    bq_servers = {'Isl': {'tabs': bq_data['tabs']}}  # 全塞给孤岛
```
- `base_isl` → 有数据（当前 flat tabs 全量）
- `base_sco` ~ `base_bob` → 空占位（正常，无数据）

## 三、存量数据迁移（Step 1，先行）

### 脚本：`_migrate_base_data.py`

```python
"""将 asa_base_quick_ref.json 从 flat 迁移到嵌套结构"""
import json

with open('asa_base_quick_ref.json') as f:
    data = json.load(f)

# 如果已经是嵌套结构，跳过
if 'servers' in data:
    print('Already nested, skip')
    exit(0)

# 迁移：flat tabs → servers.Isl.tabs
SERVERS = ['Isl','Sco','Cen','Abe','Ast','Ext','Rag','Val','Los','Gen','Bob']
existing_tabs = data.get('tabs', [])

new_data = {
    'servers': {sid: {'tabs': existing_tabs if sid == 'Isl' else []} for sid in SERVERS}
}

with open('asa_base_quick_ref.json', 'w') as f:
    json.dump(new_data, f, ensure_ascii=False, indent=2)

print(f'Migrated: {len(existing_tabs)} tabs → Isl')
```

### 迁移流程
1. SFTP 拉取 `/config/www/asa-data/asa_base_quick_ref.json`
2. 备份 `cp asa_base_quick_ref.json asa_base_quick_ref.json.bak`
3. 运行迁移脚本
4. SFTP 上传覆盖
5. 浏览器验证
6. ASA 后台「💾 保存Tab」→ build_lovelace

## 四、ASA 后台代码改动（Step 2）

### 改动 1：`loadEditorData` (L1740)

```javascript
// 改前
editorData = (data && data.tabs) ? data : {tabs:[]};

// 改后
if (mode === 'base_ref') {
  editorData = (data && data.servers) ? data : {servers:{Isl:{tabs:[]}}};
} else {
  editorData = (data && data.tabs) ? data : {tabs:[]};
}
```

### 改动 2：`currentTab` (L1728)

```javascript
// 改前
function currentTab() { return (editorData.tabs||[])[editorSelectedTab] || (editorData.tabs||[])[0]; }

// 改后
function currentTab() {
  if (editorMode === 'base_ref') {
    var srv = (editorData.servers||{})[brSelectedServer] || {};
    return (srv.tabs||[])[editorSelectedTab] || (srv.tabs||[])[0] || null;
  }
  return (editorData.tabs||[])[editorSelectedTab] || (editorData.tabs||[])[0];
}
```

### 改动 3：`renderMixedEditor` (L1750)

```javascript
// 改前
const tabs = editorData.tabs;

// 改后
const tabs = editorMode === 'base_ref'
  ? ((editorData.servers||{})[brSelectedServer]||{}).tabs || []
  : editorData.tabs;
```

### 改动 4：`addTab` (L1983)

```javascript
// 改前
editorData.tabs.push({name: name, type: 'mixed_content', content_blocks: []});

// 改后
var _tabs = editorMode === 'base_ref'
  ? ((editorData.servers||{})[brSelectedServer]||{}).tabs
  : editorData.tabs;
if (!_tabs) {
  if (editorMode === 'base_ref') {
    if (!editorData.servers) editorData.servers = {};
    if (!editorData.servers[brSelectedServer]) editorData.servers[brSelectedServer] = {};
    editorData.servers[brSelectedServer].tabs = [];
    _tabs = editorData.servers[brSelectedServer].tabs;
  } else {
    editorData.tabs = [];
    _tabs = editorData.tabs;
  }
}
_tabs.push({name: name, type: 'mixed_content', content_blocks: []});
```

### 改动 5：`rmTab` (按 same pattern)

同 `addTab` 模式，获取 `_tabs` 引用后 splice。

### 改动 6：`saveJSON` 调用不变

`saveEditor()` 保存 `editorData` 整体。嵌套结构下 `editorData = {servers:{...}}`，直接序列化即可。

### 改动 7：`saveMCBlocks` + `reRender` + 其他

通过 `currentTab()` 获取当前 tab，自动适配。无需额外改动。
