# 展开折叠方案对比

> 日期：2026-07-19 | 当前：`<details name="ic-acc">` 原生手风琴

---

## 方案1：details + CSS 动画

### 原理
保持 `<details>` + `<summary>`，通过 CSS `details[open]` 选择器驱动 `max-height` transition。

### 代码示例
```css
/* build_lovelace.py make_ic_css() 中追加 */
ha-card details.ic-acc { overflow:hidden; }
ha-card details.ic-acc .ic-body { 
  max-height:0; overflow:hidden; 
  transition:max-height 0.3s ease; 
}
ha-card details.ic-acc[open] .ic-body { max-height:2000px; }
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 改动极小（加 5 行 CSS） | ❌ `max-height` 动画不够精准（高度硬编码） |
| ✅ HTML 结构不变 | ❌ 关闭动画需要 JS 辅助获取实际高度 |
| ✅ 浏览器原生无障碍（Tab/Enter） | ❌ 手风琴仍需 JS polyfill（Android WebView） |
| ✅ 零 JS 开销 | ❌ 和现在一样无平滑关闭动画 |

### 工作量
- `build_lovelace.py`：+5 行 CSS
- `preview_server.py`：+5 行 CSS
- **总计：10 行**

## 方案2：JS onclick + CSS transition

### 原理
用 `<div>` 替换 `<details>`，`<summary>` 改为 `<div class="ic-summary">`。点击触发 `classList.toggle('open')`。CSS transition 使用真实高度（`scrollHeight`）+ `requestAnimationFrame`。

### 代码示例
```javascript
// 每个 summary 的内联 onclick
onclick="var p=this.parentElement;if(p.classList.contains('open')){p.style.maxHeight=p.scrollHeight+'px';requestAnimationFrame(function(){p.classList.remove('open');p.style.maxHeight='';});}else{p.classList.add('open');p.style.maxHeight=p.querySelector('.ic-body').scrollHeight+40+'px';}"
```

```css
ha-card .ic-acc { overflow:hidden; }
ha-card .ic-acc .ic-body { overflow:hidden; }
ha-card .ic-acc:not(.open) { max-height:28px; transition:max-height 0.3s ease; }
ha-card .ic-acc.open { max-height:none; transition:max-height 0.3s ease; }
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 展开/关闭均有平滑动画 | ❌ HTML 结构需改动（`<details>`→`<div>`） |
| ✅ 精确高度，无硬编码 | ❌ 内联 onclick 代码较长 |
| ✅ 手风琴逻辑内置，无兼容问题 | ❌ CSS class 替换 name 属性，手风琴需 JS 重写 |
| ✅ 完全可控的动画曲线 | ❌ 改动量较大 |

### 工作量
- `build_lovelace.py`：HTML 结构改动 + CSS 改动 + 手风琴 JS 重写
- `preview_server.py`：同步改动
- **总计：约 30-40 行**

---

## 对比总结

| 维度 | 方案1 (details+CSS) | 方案2 (JS+CSS) |
|------|:---:|:---:|
| 动画效果 | 🟡 展开有，关闭无 | 🟢 展开/关闭都有 |
| 改动量 | 🟢 10 行 | 🟡 30-40 行 |
| 兼容性 | 🟡 Android 需 polyfill | 🟢 全平台 |
| HTML 变动 | 🟢 无 | 🟡 改 details→div |
| 手风琴 | 🟡 依赖 name 属性 | 🟢 JS 可控 |
| 维护性 | 🟢 简单 | 🟡 中等 |

## 切换按钮形式方案

无论选方案1还是方案2，板块标题旁的展开/折叠按钮都可以有以下形式：

| # | 形式 | 说明 |
|---|------|------|
| A | ▶/▼ 三角箭头 | 最经典，CSS `transform:rotate(90deg)` 动画 |
| B | `mdi:chevron-down` 旋转 | MDI 图标，展开时旋转 180°，和三角同理 |
| C | `mdi:plus`/`mdi:minus` 切换 | 加减号，直观但无旋转动画 |
| D | 整行点击（无按钮） | 点标题行任意位置展开，最简洁 |
| E | 左侧色条 + 箭头 | 在引用色条旁显示，不占标题位置 |

> 当前用的是 `<details>` 默认三角 ▶，方案 A/C/D 可与方案1/2组合。
