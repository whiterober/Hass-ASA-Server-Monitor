# info_card 可配置展开区域 — 实施计划

> 2026-07-10 | v1203 基线

---

## 一、现状

`info_card` 块有两个模式（`collapse_descriptions` 字段）：

```
false → 全部平铺，无折叠
true  → 标题+图标在外面，所有 description 折叠在 <details> 里
```

**问题**：无法配置折叠分界线。用户希望前置几张重要的描述始终可见，其余折叠。

---

## 二、新增描述类型：`{"type": "fold"}`

插入特殊标记，渲染时以此为折叠分界线。前面内容在摘要区，后面内容在折叠区。

```json
{
  "block_type": "info_card",
  "title": "🦖 驯服速度",
  "collapse_descriptions": true,
  "descriptions": [
    {"text": "6倍"},              // ← 摘要区
    {"text": "HarvestAmount..."}, // ← 摘要区
    {"type": "fold"},             // ← 折叠分界线
    {"text": "ConfigOverride..."}, // ← 折叠区
    {"text": "ConfigOverride..."}  // ← 折叠区
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `{"type": "fold"}` | object | 折叠分界线。有 `collapse_descriptions=true` 时才生效。无此标记 = 全部折叠（旧行为） |

> 多个 `fold` 标记：只认第一个。`br` / `icon_group` / `fold` 都不计入可见行数。

---

## 三、渲染改动

### 改前

```
标题+图标          ← 始终可见
<details>
  <summary>···</summary>
  描述0,1,2,...    ← 全部在折叠区
  <span>···</span>
</details>
```

### 改后（有 fold 标记）

```
标题+图标          ← 始终可见
描述0, 描述1       ← 始终可见（摘要区，到 fold 为止）
<details>
  <summary>···</summary>
  描述3, 描述4,...  ← 折叠区（fold 之后）
  <span>···</span>
</details>
```

渲染逻辑：遇到 `{"type": "fold"}` 即闭合摘要区，后续全部进 `<details>`。

## 五、ASA 后台编辑器改动

### 5.1 新增「+ 折叠区」按钮

在 `+ 图标组` 按钮同排加一个，样式同为 `btn-small btn-primary`。点击插入 `{"type": "fold"}`。编辑器中显示虚线 "──── 折叠线 ────"。

### 5.1b 顺手修复按钮间距

描述行按钮行 `gap:8px`，板块级 `gap:4px`，不一致。统一为 `gap:4px`。

```
[...] [+ 描述] [+ 换行] [+ 图标组]  [⋯]  [前N条可见: 2 ▼]
                                     ↑折叠按钮              ↑新增
```

HTML 插入点：`mcICCollapseDescs` checkbox 之后（L3041 附近）

```html
<input type="number" id="mcICCollapseAfter'+bi+'" 
  value="'+(b.collapse_after||'')+'" 
  min="0" max="99" placeholder="全部折叠"
  style="width:48px;height:24px;font-size:0.7em;border-radius:4px;border:1px solid var(--border);
  display:'+(b.collapse_descriptions?'inline-block':'none')+'"
  title="前N条始终可见，其余折叠">
```

折叠按钮 onclick 加一行控制显隐：
```javascript
document.getElementById('mcICCollapseAfter'+bi).style.display = a ? 'inline-block' : 'none';
```

### 5.2 保存

saveMCBlocks 遍历描述时识别 fold 类型 DOM 元素，输出 `{"type": "fold"}`。

### 5.3 渲染

renderMC 已有 `{"type": "br"}` 渲染。加同模式 `{"type": "fold"}` —— 显示折叠标记线。
