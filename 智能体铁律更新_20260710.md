## 🔗 跨会话 Lovelace 安全修改铁律（2026-07-10 解耦后更新）

> 以下规则来自 `build_lovelace.py` 路径白名单解耦改造。改造后，`build_lovelace.py` 仅管理 13 个从 JSON 重建的视图，非 ASA 视图完全不受 build 影响。

### 核心变更

```
改造前：build_lovelace.py 丢弃所有索引 ≥ 6 的视图 → 非 ASA 手动页面必丢
改造后：build_lovelace.py 按路径白名单识别，非 ASA 视图原样保留 → 永不丢失
```

### ASA 管理边界（`ASA_BUILD_PATHS`，共 13 个）

```python
ASA_BUILD_PATHS = {
    'asa-server-rules',     # 服务器规则 ← server_rules.json
    'info_whiterober',      # 部落运维速查 ← tribe_ops.json
    'base_isl', 'base_sco', 'base_cen', 'base_abe',
    'base_ast', 'base_ext', 'base_rag', 'base_val',
    'base_los', 'base_gen', 'base_bob',  # 基地速查 ×11 ← asa_base_quick_ref.json
}
```

> ⚠️ 以下视图**不在 ASA 管理范围**（build 原样保留）：家、电视遥控器、工作室、方舟、服务器操作、资讯广场、快捷输入、info_whiterober_old、base_whiterober_old、以及任何用户手动新增的页面。

### 铁律：非 ASA 页面手动修改无需 build

- **非 ASA 页面**（所有不在 `ASA_BUILD_PATHS` 中的视图）：手动修改 lovelace JSON 后**完全不需要**执行 `build_lovelace.py`。即使有人误触「保存Tab」触发 build，修改也不会丢失。
- **ASA 页面**（13 个 `ASA_BUILD_PATHS` 视图）：不可手动改 lovelace JSON。应通过 ASA 后台 UI 编辑 → 保存Tab → build 正常流程。

### 铁律：另一个会话改 lovelace 后

- 另一个会话（非本 ASA 智能体）修改 lovelace 并 SFTP 上传后：
  - 如果改的是**非 ASA 页面**：无需任何操作
  - 如果改的是**ASA 页面**：必须 SSH 执行 `python3 /config/build_lovelace.py`
- `build_lovelace.py` 已自动同步 `.storage/lovelace.lovelace`，**无需额外重启 HA**
- **禁止跳过此步骤**：跳过会导致 ASA 视图停留在上传时的旧版本
