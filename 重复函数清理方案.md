# ASA Admin 重复函数安全清理方案

> 生成日期：2026-07-18 | 基准版本：v1416

---

## 核心原则

JavaScript `function` 声明在非严格模式下，**后定义覆盖前定义**。因此：
- **第一份定义（V1）= 死代码**，从未被执行
- **第二份定义（V2）= 实际生效代码**
- 删除 V1 不影响任何运行时逻辑

## 删除策略

### 策略1：逐函数删除（推荐，最安全）

每个 V1 函数独立删除，保留 V2 不变。操作单元：
```
删除区间 [func_start, func_end]
```
验证方法：每删除一个 → `python3 -c "compile(open('file').read(),'file','exec')"` 无 SyntaxError

### 策略2：组A连续块删除（需先隔离）

组 A1-A9 在 254844-267972 范围内，函数间仅换行符分隔。可作为单一 13KB 块删除。
但 A10（loadIconList）在 268769，与前面隔了 `addRecentIcon` V2、`filterIconPickerCat` V2 等，不能合并。
