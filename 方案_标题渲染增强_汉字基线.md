## 问题 a：板块级标题渲染能力

### 现状

`ic_title`（卡片标题，L1857）当前仅调用了 `_render_mdi_inline()`：

```python
parts.append('<span{}>{}</span>'.format(title_color_style, _render_mdi_inline(ic_title, auto_cls)))
```

### 对比描述行（`.ic-text`）的能力清单

| 能力 | 描述行 | ig 标题 | **ic_title（板块级）** |
|------|:---:|:---:|:---:|
| `mdi:xxx` → `<ha-icon>` | ✅ `_render_mdi_inline` | ✅ | ✅ |
| 首位 `mdi:xxx` 提取为前置图标 | ✅ L2009-2013 | ✅ L1915-1918 | ❌ 缺失 |
| `^` 序号自动编号 | ✅ L1964-1966 | ✅ L1927 | ❌ 缺失 |
| `[xxx]` → badge 标签 | ✅ `_render_badges` | ✅ L1933-1935 | ❌ 缺失 |

### 方案

L1857 替换为：

```python
# 板块级标题：提取首位 mdi 为前置图标 + mdi 渲染 + ^ 序号 + [] 徽章
_ic_title_mdi = re.match(r'mdi:([\w-]+)', ic_title)
if _ic_title_mdi:
    _title_icon = '<ha-icon icon="mdi:{}" style="color:inherit;--mdc-icon-size:16px;width:16px;height:16px;margin-right:4px;flex-shrink:0"></ha-icon>'.format(_ic_title_mdi.group(1))
    _title_text = ic_title[_ic_title_mdi.end():].strip()
else:
    _title_icon = ''
    _title_text = ic_title
# ^ 序号
_is_hat = _title_text.startswith('^')
if _is_hat:
    _title_text = _title_text[1:].lstrip()
    # 板块级标题固定一个序号（不分 block/linear）
    _title_text = '<span class="ic-badge ic-badge-num">1</span> ' + _render_badges(_render_mdi_inline(_title_text), False)
else:
    _title_text = _render_badges(_render_mdi_inline(_title_text), False)
parts.append('<span{}>{}{}</span>'.format(title_color_style, _title_icon, _title_text))
```

**涉及改动**：1 处 HTML（L1857 替换）。

## 问题 b：全局汉字比英文高 1px

### 根因

Roboto（拉丁字体）和系统 CJK 回退字体的**基线位置不同**。拉丁字体的 x-height 居中，汉字是方块字填充整格。在相同 `font-size` / `line-height` 下，汉字视觉重心比英文高约 1px。

### 方案

在 `make_ic_css()` 开头追加 1 条 CSS：

```css
/* 全局汉字基线修正：CJK 字符上移 1px 使视觉中心与拉丁对齐 */
ha-card .info-card-block *{text-underline-position:under}
```

❌ 但 `text-underline-position` 只影响下划线位置，不影响文字本身。

实际可行的 CSS 方案：

```css
ha-card .info-card-block,
ha-card .info-card-block * {
  font-family: 'HarmonyOS Sans SC', 'Roboto', 'Noto Sans SC', sans-serif !important;
}
```

如果 HA 服务器装了 HarmonyOS Sans SC 或 Noto Sans SC，这些字体对 CJK+Latin 混排有优化的基线对齐。**前提：HA 服务器需要安装该字体**。

若不想依赖字体安装，纯 CSS 方案：

```css
/* 对含 CJK 字符的行应用微调 */
ha-card .info-card-block .ic-text,
ha-card .info-card-block .ic-title {
  line-height: 1.4 !important;
}
```

但这是全局改行高，副作用较大。

**最保守方案**：不做全局修改，因为 1px 偏差在不同字体/浏览器/操作系统下表现不一致，强制修正可能在某些环境下反而更差。建议仅在用户确认具体字体环境后按需调整。
