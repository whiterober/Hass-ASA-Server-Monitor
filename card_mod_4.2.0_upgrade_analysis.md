# Card-mod 4.2.0 升级影响分析

> 分析日期：2026-06-09
> 目标版本：card-mod 4.2.0
> 要求 HA 版本：2026.2.0+

---

## 📊 代码扫描结果

### card_mod 使用统计

| 文件 | 出现次数 | 使用模式 |
|------|---------|---------|
| `lovelace` | **119 处** | 全部为 `"card_mod": {"style": "..."}` |

### card_mod 使用分类

| 类型 | 数量 | 风险 |
|------|------|------|
| 静态 style（纯 CSS） | ~105 处 | 🟢 无影响 |
| 动态 style（含 Jinja2 模板） | ~3 处 | 🟡 需关注 |
| 在线编辑器内 style | ~11 处 | 🟢 仅 UI 修复 |
| `{# card_mod.debug #}` | 0 处 | 🟢 无影响 |
| `card-mod-<thing>-yaml` 主题 | 待确认 | 🔴 见下文 |

## 🔍 逐项影响分析

### 1. HA 版本要求：2026.2.0+ 🔴 必须确认

**影响**：如果当前 HA 版本低于 2026.2.0，安装 4.2.0 将导致 card-mod **完全无法加载**。

**检查方法**：
- HA 界面 → 设置 → 关于 → 查看版本号
- 或开发者工具 → 信息 → `version`

**结论**：升级前必须确认 HA ≥ 2026.2.0。

---

### 2. 主题 `card-mod-<thing>-yaml` 段 (#571) 🔴 需手动检查

**变更**：4.2.0 修复了 HA 2026.2.0 下使用 `card-mod-<thing>-yaml` 主题段时 card-mod 加载失败的问题。

**影响**：如果 `themes/` 目录下的任何主题文件包含类似以下字段：

```yaml
card-mod-card-yaml: |
  ha-card { ... }
card-mod-view-yaml: |
  ...
card-mod-badge-yaml: |
  ...
```

**当前状态**：工作区中无 `themes/` 目录（HA 配置通过 `!include_dir_merge_named themes` 引入），无法直接扫描。

**检查方法**：
- 在 HA 配置目录中检查 `themes/` 下所有 `.yaml` 文件
- 搜索 `card-mod-`、`card_mod-` 关键词

**结论**：如果使用了 `card-mod-<thing>-yaml` 主题变量，4.2.0 修复了兼容性问题，升级是**必要**的。如果没用，无影响。

---

### 3. 动态模板 style 🟡 低风险

**变更**：4.0.0 起模板 debug 输出改为对象格式。4.2.0 没有进一步改动。

**使用位置**（3 处动态模板 style）：

| lovelace 行号 | 功能 | 模板内容 |
|--------------|------|---------|
| L2192 | 服务器弹窗标题背景色 | `{% set selected_server = states(...) %}` + `{{ theme.get('primary_color') }}` |
| L2504 | 玩家按钮列表（auto-entities） | `{% for e in states.input_button ... %}` + `{% if loop.last %}` |
| L~2121 | 服务器卡片动态主题色 | 类似 L2192 的模式 |

**分析**：
- 这些模板**没有使用** `{# card_mod.debug #}` 注释，不受 debug 输出格式影响
- Jinja2 模板内联在 `style` 字符串中，card-mod 在渲染前执行模板求值
- 4.2.0 **没有改变模板求值机制**，不会影响动态样式渲染

**结论**：🟢 不受影响。

---

### 4. 子元素样式修复 (#554) 🟢 改进

**变更**：修复了当没有主 `style` 时子元素样式不生效的问题。

**分析**：代码中绝大多数 `card_mod` 都有明确的 `style` 属性，不依赖此 bug。即使有边缘 case，这属于 **bug 修复**，行为更正确而非破坏。

**结论**：🟢 不受影响，甚至可能修复了之前偶发的样式不生效问题。

---

### 5. 延迟渲染卡片修复 🟢 改进

**变更**：对延迟渲染的卡片，自动应用 `prepend` 选项。

**分析**：部分自定义卡片（如 `custom:button-card`、`custom:auto-entities`）可能存在延迟渲染。4.2.0 会自动检测并修正注入时机，确保样式生效。

**结论**：🟢 改进项，可能修复某些卡片样式偶尔不生效的问题。

---

### 6. DOM 断开样式丢失修复 (#563) 🟢 改进

**变更**：修复了元素断开 DOM 连接时主题样式丢失的偶发问题。

**结论**：🟢 改进项，提升样式稳定性。

---

### 7. 编辑器内 card_mod 图标修复 🟢 无关

**变更**：修复编辑器中 card_mod 样式图标显示问题。

**结论**：🟢 仅影响 HA 前端编辑器视图，不影响实际渲染。

---

### 8. `extra_module_url` 提醒 ⚠️ 需确认

**变更日志提醒**：如果使用 HACS 和 `extra_module_url`，需要更新 URL 以匹配 Dashboard resource。

**当前状态**：`configuration.yaml` 中 `frontend:` 仅配置了 `themes`，**未使用** `extra_module_url`。

**结论**：🟢 不受影响。

---

### 9. ha-assist-chip 样式 🔵 新功能

**变更**：新增对 `ha-assist-chip` 的 card_mod 支持。

**分析**：纯粹的新增功能，不影响现有卡片。如果将来在 entities card header/footer 中使用按钮，可以利用此功能。

**结论**：🟢 无影响（可选新功能）。

---

### 10. 重复补丁警告移至日志 (#582, #588, #589) 🟢 改进

**变更**：重复补丁警告从浏览器通知改为仅写入 HA 日志。

**结论**：🟢 改进项，减少不必要的浏览器弹窗。

| # | 检查项 | 状态 |
|---|--------|------|
| 1 | HA 版本 ≥ 2026.2.0 | ⬜ 确认 |
| 2 | `themes/` 中无 `card-mod-<thing>-yaml`（或有则受益于修复） | ⬜ 检查 |
| 3 | `configuration.yaml` 无 `extra_module_url` | ✅ 已确认 |
| 4 | lovelace 中无 `{# card_mod.debug #}` 模板注释 | ✅ 已确认 |

## 🎯 结论

**card-mod 4.2.0 升级对现有代码的风险评级：🟢 低风险**

1. **唯一的硬性门槛**是 HA 版本 ≥ 2026.2.0，确认后即可升级
2. 所有 119 处 `card_mod` 使用均为标准 `style` 注入模式，未被 4.2.0 的任何破坏性变更影响
3. 4.2.0 包含的多项 bug 修复（子元素样式、延迟渲染、DOM 断开）可能**改善**现有样式的稳定性
4. 如果 `themes/` 中使用了 `card-mod-<thing>-yaml`，4.2.0 的 #571 修复是**必须的**

### 建议操作顺序

1. 确认 HA 版本 ≥ 2026.2.0
2. 检查 `themes/` 中是否有 card-mod 主题变量
3. 升级 card-mod 到 4.2.0
4. 重启 HA
5. 检查浏览器控制台无 card-mod 相关错误
6. 抽查几个使用 card_mod 的仪表板卡片样式正常
