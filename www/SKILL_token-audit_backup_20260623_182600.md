# Token Audit (Lightweight)

## 目标
- 在不回读全量历史的前提下，快速评估 token 风险与成本趋势。
- 审计本身必须低开销、可复用、可快速终止。


## 触发条件
- **强制**：每轮第一步执行 Session Memory Token 追踪（静默读写，无输出）
- 仅达到阈值时向用户输出警告
- 用户主动提到 token/计费/成本时输出详细审计


## 预警与高风险弹窗机制
- 提前预警：当本轮 token 消耗、累计 token 数、上下文长度等接近高风险阈值（如达到80%），自动调用 #vscode/askQuestions 弹窗提醒用户，提示"已接近高风险区，建议提前优化"。
- 高风险弹窗：仅当实际达到高风险阈值时，再弹窗提示"已超标"，并给出最小动作建议。
- 低/中风险仅写进度文件，不弹窗，避免频繁打扰。
- 弹窗内容包含：风险等级（预警/高风险）、触发原因、建议动作，允许用户关闭或调整阈值。
- 同一会话内仅首次预警/高风险分别触发弹窗，后续不重复弹出，除非用户主动调整设置。
- 支持自定义阈值（如单轮 token 数、累计 token 数、模型倍率等），默认采用 skill 内置建议。

## 最小流程（最多 4 步）
1. 识别范围：仅看最近 N 轮，默认 6-10 轮。
2. 聚合统计：复用已有计数/趋势，不做全文回读。
3. 风险判定：输出低/中/高并给 1 条最小措施。
4. 摘要输出：3-6 行，包含结论、依据、动作。

## 输出约束
- 固定 3-6 行。
- 每行仅 1 个信息点。
- 禁止长段落、表格、逐轮详单。

## 强制禁令
- 禁止全文回读（除非用户明确要求并同意更高开销）。
- 禁止重复扫描同一批消息。
- 禁止重复计算同一轮统计。
- 禁止把轻量审计升级为重型分析。

## 轮次阈值建议
- 建议阈值：30-50 轮（可按团队套餐与任务复杂度调整）。
- <30 轮：按需触发。
- 30-50 轮：每轮在结论输出后执行轻量审计。
- >50 轮：进入强省 token 模式（更短回答、减少重述）。



## 风险分级
- 低：最近 N 轮稳定、重复率低。
- 中：长度上升或重复解释增加。
- 预警：消耗接近高风险阈值（如80%），提前弹窗提醒。
- 高：长度持续上升且重复信息明显，已超高风险阈值。
- 高风险最小动作：下一轮仅保留结论+必要步骤，背景压缩为 1 行。
- 预警/高风险时自动弹窗，低/中风险仅进度记录。

## 🤖 Session Memory Token 追踪器（强制，不可跳过）

### 原理
用 `/memories/session/token-tracker.json` 持久化累加 token 消耗，不受上下文长度影响，确保在对话临界点前可靠预警。

### 文件结构
```json
{
  "total": 48500,
  "max": 180000,
  "warnAt": 120000,
  "criticalAt": 150000,
  "lastTurn": 62,
  "history": [3800, 4200, 2800]
}
```

### 每轮 SOP（第一步执行，不可跳过）
1. **读** `/memories/session/token-tracker.json`（若不存在则初始化 total=0）
2. **算**：user 输入字符数 + 本答案字符数，各 ÷ 2.5 估算 token
3. **加**：total += 本轮估算 token
4. **写**：保存回 session memory（不输出到用户）
5. **判**：
   - total < warnAt → 静默，不输出任何警告
   - total >= criticalAt → 🔴 "对话已达临界点，强烈建议开启新对话"
   - total >= warnAt → 🟡 "对话接近临界点，规范遵守度可能下降"

### 通用性
- 计数器不绑定任何项目，跟踪整个会话
- 阈值首次初始化时可调整
- 跨项目、跨 workspace 复用

### 副作用
- 每轮计数开销约 200-350 token，正常回合占比 5%-10%
- 低于一次"乱改-回退"循环的浪费，净收益为正

## 与进度协同
- 与 progress-tracking 并行，不互相触发全文回读。
- 审计只评估开销，不改变审批状态机。

## 进度落盘
- 每轮在结论输出后，将审计摘要追加到当前进度文件（1-2 行）。
- 推荐格式：`Token审计：风险=<低|中|高>；依据=<1条>；动作=<1条最小措施>`。
- 仅写摘要，不写长分析。

## 数据边界
- 只使用当前会话可直接获得的数据和已有聚合结果。
- 无聚合数据时只做趋势判断，不补做重型统计。
- 同一轮只计算一次风险等级，变化小可复用上轮结论。

## 终止条件
- 达到 3-6 行且已包含风险等级与措施时立即停止。
- 用户未要求细节时不扩写背景。

## 执行清单（结论输出后）
- [ ] 已执行轻量审计
- [ ] 范围限定为最近 N 轮（默认 6-10）
- [ ] 未全文回读，且无重复扫描/重复计算
- [ ] 已给出风险分级、预警与高风险最小动作
- [ ] 预警/高风险时已自动弹窗（仅首次），低/中风险仅进度记录
- [ ] 审计输出控制在 3-6 行
- [ ] 已将审计摘要（1-2 行）追加到同一进度文件
