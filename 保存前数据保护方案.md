# ASA 后台保存前数据丢失——综合治理方案

> 日期：2026-07-19 | 基准版本：v1442

---

## 根因分析

### 数据流
```
用户操作 → editorData (内存) → 保存TAB → saveEditor() → server_rules.json (磁盘)
                                                    ↓
                                            build_lovelace.py
```

### 丢失场景

| # | 场景 | 代码位置 | 影响 |
|---|------|---------|------|
| 1 | **切换菜单**（服务器规则↔部落运维↔部落基地） | L1178-1180：`nav button click` → `loadEditorData()` | 🔴 全部未保存更改丢失 |
| 2 | 页面重载 | 浏览器重载 → `loadEditorData()` | 🔴 同场景1 |
| 3 | 同菜单内切 TAB | `reRender()` 仅重绘，不重载数据 | ✅ 安全 |

### 核心代码

```javascript
// L1170-1180：菜单切换按钮
btn.addEventListener('click', () => {
    const panel = btn.dataset.panel;
    if (panel === 'server-rules') loadEditorData('server_rules');  // ← 直接重载
    else if (panel === 'tribe-ops') loadEditorData('tribe_ops');
    else if (panel === 'base-ref') loadEditorData('base_ref');
});

// L1589-1597：loadEditorData — 全量替换 editorData
async function loadEditorData(mode) {
    const data = await loadJSON(cfg.file);   // ← 从服务器重新拉取
    editorData = (data && data.tabs) ? data : {tabs:[]};  // ← 覆盖内存数据
    editorSelectedTab = 0;
    // ...
}
```

## 方案设计

### 策略：离开前提示保存

在菜单切换和页面离开前，检测是否有未保存的更改，弹窗提示用户**「有未保存的更改，是否保存？」**。

### 实现

#### 1. 脏标记（dirty flag）

```javascript
var _editorDirty = false; // 任何增删改操作设为 true，保存后设为 false

// 在以下操作中设 _editorDirty = true：
// - addTab(), rmTab()
// - addMCBlock(), removeBlock()
// - addICDesc(), removeICDesc(), addICBr(), addICFold(), addICCopyKey()
// - 任何 description 文字修改（oninput 事件）
// - duplicateBlock(), pasteBlock(), pasteDesc()
// - copyTab(), pasteTab()

// 保存时清除：
function saveEditor() {
    // ... existing save logic ...
    _editorDirty = false;
}
```

#### 2. 拦截菜单切换

```javascript
// L1170：改造 nav button click
btn.addEventListener('click', () => {
    const panel = btn.dataset.panel;
    const targetMode = panel === 'server-rules' ? 'server_rules' :
                       panel === 'tribe-ops' ? 'tribe_ops' : 'base_ref';
    
    if (_editorDirty && targetMode !== editorMode) {
        showConfirm('有未保存的更改，是否保存后再切换？', [
            {text: '保存并切换', action: () => { saveEditor().then(() => loadEditorData(targetMode)); }},
            {text: '不保存直接切换', action: () => { _editorDirty = false; loadEditorData(targetMode); }},
            {text: '取消', action: () => {}}
        ]);
    } else {
        if (targetMode !== editorMode) loadEditorData(targetMode);
    }
});
```

#### 3. 拦截页面关闭/刷新

```javascript
// 浏览器 beforeunload 事件
window.addEventListener('beforeunload', (e) => {
    if (_editorDirty) {
        e.preventDefault();
        e.returnValue = '有未保存的更改，确定离开？';
    }
});
```

#### 4. 脏标记设置点（需修改的函数）

| 函数 | 位置 | 操作 |
|------|------|------|
| `addTab()` | L1875 | `_editorDirty = true` |
| `rmTab()` | L5265 | `_editorDirty = true` |
| `addMCBlock()` | L2944 | `_editorDirty = true` |
| `pasteBlock()` | L2931 | `_editorDirty = true` |
| `duplicateBlock()` | L2924 | `_editorDirty = true` |
| `removeBlock(btn)` | 搜索 | `_editorDirty = true` |
| `addICDesc()` | 搜索 | `_editorDirty = true` |
| `copyTab()` | L1891 | `_editorDirty = true`（仅黏贴时） |
| `pasteTab()` | L1898 | `_editorDirty = true` |
| `reRender()` | 搜索 | 标题/描述文字 oninput 设为 dirty |

**总量估算**：修改约 12 个函数 + 2 处事件监听 + 1 个全局变量 ≈ 50 行新增代码。

## 补充分析：增删改操作对数据的影响

### 板块级操作

| 操作 | 函数 | 数据影响 | 风险 |
|------|------|---------|------|
| **添加卡片** | `addMCBlock()` L2978 | 推送默认空块 → `reRender()` → normalize 循环 | 🟡 空 `images[0].image_url=''` 覆盖其他卡片 `icon_url`（Bug 3 已修） |
| **复制卡片** | `duplicateBlock()` L2924 | 深拷贝到 `_blockClipboard` | ✅ 无影响 |
| **黏贴卡片** | `pasteBlock()` L2931 | 深拷贝推入 → `reRender()` | ✅ 安全 |
| **删除卡片** | `removeBlock()` | DOM 移除 + editorData splice | 🟡 未保存前删除，数据永久丢失 |
| **保存卡片** | `saveMCBlocks()` L3076 | **从 DOM 逐字段读回 editorData** | 🔴 `b.icon_url = document...value \|\| ''` — 空输入覆盖已有值 |
| **渲染** | `renderMixedContent()` L2309 | normalize 循环 | 🔴 `b.icon_url = b.images[0].image_url` 覆盖（Bug 3） |

### 行级操作

| 操作 | 函数 | 数据影响 | 风险 |
|------|------|---------|------|
| **添加描述** | `addICDesc()` L2790 | 推送默认空描述 | ✅ 不影响已有 |
| **删除描述** | `removeICDesc()` L2739 | `splice(idx,1)` + DOM 重索引 | 🟡 删除不可恢复 |
| **保存描述** | `saveMCBlocks()` L3161 | 逐字段 DOM→editorData | 🟡 空 DOM 值覆盖已有（`image_url`, `quantity`, `color`, `opacity`, `server` 等） |
| **添加图标组** | `addICIconGroup()` | 推送空 icon_group | ✅ 安全 |
| **渲染 normalize** | L2315-2317 | `d.image_url = d.images[0].image_url` | 🟡 覆盖行级 image_url |
| **渲染 normalize** | L2316 | `d.quantity = d.images[0].quantity` | 🟡 覆盖行级 quantity |

### 具体数据清空风险矩阵

| 字段 | 操作时清空风险 | 根因 |
|------|:---:|------|
| `b.icon_url` | 🔴 | normalize 用 `images[0].image_url` 覆盖；save 用 DOM 空值覆盖 |
| `b.title` | 🟡 | save 用 DOM input 覆盖 |
| `d.image_url` | 🔴 | normalize 用 `d.images[0].image_url` 覆盖 |
| `d.quantity` | 🟡 | normalize 用 `d.images[0].quantity` 覆盖 |
| `d.color` | 🟡 | save 默认 `'auto'` |
| `d.opacity` | 🟡 | save 默认 `1.0` |
| `d.server` | 🟡 | save 默认 `''` |
| `d.server_states` | 🟡 | save 默认 `{}` |
| `b.quote_color` | 🟡 | save 默认 `'#FF9800'` |
| `b.server_states` | 🟢 | save 从 DOM JSON 读取 |

### 建议防护措施

1. **保存前 DOM→editorData 同步**：在 `reRender()` 前调用 `saveMCBlocks()` 将当前 DOM 状态写回 editorData
2. **空值保护**：save 时检查 DOM 值是否为空，为空则保留 editorData 原值
3. **脏标记**：所有增删改操作设 `_editorDirty = true`（同主方案）

## 追加 Bug：行级图片黏贴后丢失（与 Bug 3 同根因）

### 复现
1. 给行级元素选图片
2. 黏贴（或添加）新行级元素
3. 刚才选的图片消失

### 根因

`selectDescImage()` L5675 — 完全没写 `editorData`：
- 只设 DOM 隐式输入 `mcICDescImg{bi}_{di}.value = url`
- 创建 `d.images[0] = {image_url: '', ...}`（L5829）用于颜色模式

渲染 normalize 循环 L2315：
```javascript
d.image_url = d.images[0].image_url;  // 覆盖为空！
```

### 修复点（selectDescImage）
1. 设图片后追加：`d.image_url = url`
2. `d.images[0].image_url: ''` → `image_url: url`

| 风险等级 | 场景 | 说明 |
|:---:|------|------|

## 统一解决方案

### 三层防护体系

```
┌─────────────────────────────────────────┐
│ 第1层：离开拦截（菜单切换/页面关闭）      │
│ _editorDirty → 弹窗确认 → 保存/丢弃/取消   │
├─────────────────────────────────────────┤
│ 第2层：操作安全（增删改黏贴前 DOM→存档）   │
│ reRenderGuard → saveMCBlocks → reRender  │
├─────────────────────────────────────────┤
│ 第3层：字段保护（空值不覆盖已有数据）       │
│ save 时检查 DOM 值 → 空则保留 editorData   │
└─────────────────────────────────────────┘
```

### 第1层：离开拦截

| 触发场景 | 拦截方式 | 代码位置 |
|---------|---------|---------|
| 切换菜单 | `_editorDirty` → `showConfirm` 三选 | L1170 改造 |
| 页面关闭/刷新 | `beforeunload` 事件 | 新增 |
| 保存完成 | `_editorDirty = false` | `saveEditor()` 末尾 |

### 第2层：`reRenderGuard` 包装器

```javascript
// 所有调用 reRender() 的地方改为 safeReRender()
function safeReRender() {
  var tab = editorData.tabs[editorSelectedTab];
  if (tab) saveMCBlocks(tab);  // DOM → editorData 同步
  reRender();
}
```

需要替换的调用点：
- `addMCBlock` → `safeReRender()`
- `pasteBlock` → `safeReRender()`
- `pasteDesc` → `safeReRender()`
- `addICDesc/removeICDesc/addICBr/addICFold/addICCopyKey/addICIconGroup` → `safeReRender()`
- `addTab/rmTab` → `safeReRender()`
- `copyTab/pasteTab` → `safeReRender()`
- `selectDescImage`（图标组模式）→ `safeReRender()`

### 第3层：字段保护（saveMCBlocks 改造）

| 字段 | 当前 | 改造后 |
|------|------|--------|
| `b.icon_url` | `= dom.value \|\| ''` | `= dom.value \|\| b.icon_url` |
| `b.quote_color` | `= dom.value \|\| '#FF9800'` | `= dom.value \|\| b.quote_color` |
| `b.title_color` | `= dom.value \|\| 'auto'` | `= dom.value \|\| b.title_color` |
| `d.image_url` | `= dom.value \|\| ''` | `= dom.value \|\| d.image_url` |
| `d.color` | `= dom.value \|\| 'auto'` | `= dom.value \|\| d.color` |
| `d.opacity` | `= val \|\| 1.0` | `= dom.value ? val : d.opacity` |
| `d.server` | `= dom.value \|\| ''` | `= dom.value \|\| d.server` |

> 核心原则：DOM 有值→用 DOM；DOM 空→保留 editorData 原值，不覆盖。
