# 通用进度追踪 SKILL

## 概述
此 SKILL 适用于任意项目的进度记录。唯一顺序口径：先向用户输出结论，再明确告知"将开始写进度"，随后写进度；写后不再输出任何结论。同一会话复用同一文件，支持跨设备续接。

## 核心规则

### 0. 防回归目标（强制）
- 必须写进度：每回合完成结论输出后，必须完成一次进度写入（创建或追加）。
- 禁止按回合新建：同一会话内，不得因新回合消息创建新文件。
- 禁止全文回读：默认不得读取目标文件全文（仅用户明确要求时例外）。
- 无读取优先（强制）：默认先尝试不读取进度文件直接末尾追加。
- 仅读最后一条：只允许读取"最后一条进度块"所需的尾部少量行定位续写点。
- 一律末尾追加：新进度必须直接追加到文件末尾，不在中间插入改写。
- 历史检索受限：只有用户明确要求时，才允许回读更早历史内容。
- 缺失即重试：写入失败先重试或报错，再回复用户。
- 顺序口径（唯一）：先给用户结论（变更点/结果/验证），再写进度。
- 写前告知（强制）：开始写进度前，必须先告知用户"接下来将写进度"。
- 写后不二次报告：完成进度写入后，不再向用户重复同一结论。
- 写后禁结论输出（强制）：完成进度写入后，不允许再向用户输出任何结论性内容。

### 0.1 优先级说明（冲突处理）
- 若与当前会话中更高优先级的运行指令冲突，必须以更高优先级指令为准。
- 发生冲突时，先向用户说明冲突原因，再给出可执行替代流程（最小可落地步骤）。

### 1. 文件组织
- 存储位置：`<workspace>/进度/`（若项目另有约定，以项目规范为准）。
- 命名格式：`<序号>_<任务类型>_<简述>.md`。
- 序号规则：3 位数字递增（001, 002, 003...）。
- 会话复用（强制）：同一会话始终复用同一文件。
- 新建触发（满足其一）：
  1. 用户明确新开会话/新任务。
  2. 用户明确要求新建进度文件。
  3. 当前会话已完结且用户确认进入新会话。

### 2. 命名示例
| 任务类型 | 文件名示例 |
|---|---|
| 开发 | `001_开发_新增接口鉴权.md` |
| 发布 | `002_发布_v1.2.0上线.md` |
| 本地化 | `003_本地化_补充英文文案.md` |
| 日志诊断 | `004_日志诊断_启动异常排查.md` |
| 迁移 | `005_迁移_工作区路径迁移.md` |
| 系统配置 | `006_系统配置_更新审批规则.md` |

### 3. 记录模板
```markdown
# 对话进度 - <任务标题>

## 基础信息
- 对话序号：#<N>
- 创建时间：<YYYY-MM-DD HH:MM:SS>
- 任务类型：<开发|发布|本地化|日志诊断|迁移|系统配置|其他>
- 用户需求：<简述>

## 执行步骤与进度
### 步骤 1：<步骤名称>
- 时间：<YYYY-MM-DD HH:MM:SS>
- 状态：✅ 完成 / ⏳ 进行中 / ❌ 失败
- 描述：<本步内容>

## 最终结果
- 整体状态：✅ 成功 / ❌ 失败
- 关键输出：<文件/配置/结果>
- 待处理事项：<下一步>
- 跨平台备注：<无则写 无>
```

### 4. 更新方式
- 尾追加策略（强制）：定位活动文件后，直接追加到末尾。
- 零读取优先：优先使用"已知尾锚点"直接追加，不先读取进度文件内容。
- 失败回退：仅当零读取追加失败时，才允许最小尾读（建议 20-80 行）后重试追加。
- 最小读取：默认仅读取尾部少量行；禁止全文回读（除非用户明确要求）。
- 续写定位：仅基于最后一条进度块定位续写点，定位后直接末尾追加。
- token 审计协同：若本回合执行 token-audit，追加 1-2 行摘要（风险/依据/动作）。

### 5. 防遗忘清单（收尾阶段）
1. 先输出本回合结论（变更点/结果/验证）。
2. 明确告知用户"接下来将写进度"。
3. 确认目标进度文件。
4. 先按"无读取优先"直接末尾追加本回合新增步骤/决策/结果。
5. 若追加失败，再最小尾读（20-80 行）定位续写点后重试。
6. 校验写入成功（优先看文件存在、时间、大小，必要时抽查末尾）。
7. 若触发 token-audit，追加 1-2 行审计摘要。
8. 写完即结束本回合，不再输出任何结论性内容。

## 推荐锚点
- 首次写入后，在 `/memories/session/active_progress_file.md` 记录活动文件。
- 后续回合优先读取锚点，防止误新建。

## 适用场景
- 任意开发、发布、诊断、迁移、配置类任务。
- 多智能体协作场景。
- 需要跨设备继续的会话任务。
